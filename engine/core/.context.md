# engine/core — System Context

## System Purpose

The core subsystem is the foundation of FastFreeEngine. It provides the application lifecycle (window creation, main loop, shutdown), an Entity Component System (ECS) for organizing game objects, a per-frame arena allocator for fast temporary memory, a lightweight error type that replaces exceptions, structured logging, hardware tier detection, and fixed-width type aliases used everywhere in the engine. If you are making a game with FFE, every piece of your code depends on this subsystem.

---

## Public API

### Type Aliases (`core/types.h`, namespace `ffe`)

| Alias | Underlying Type | Notes |
|-------|----------------|-------|
| `i8`, `i16`, `i32`, `i64` | `int8_t` ... `int64_t` | Signed integers |
| `u8`, `u16`, `u32`, `u64` | `uint8_t` ... `uint64_t` | Unsigned integers |
| `f32` | `float` | 32-bit float |
| `f64` | `double` | 64-bit float |
| `EntityId` | `u32` | Opaque entity handle. Upper 12 bits = generation, lower 20 bits = index |
| `ComponentId` | `u32` | Runtime type ID for Lua bindings (not RTTI) |

**Constants:**

| Name | Type | Value | Description |
|------|------|-------|-------------|
| `NULL_ENTITY` | `EntityId` | `~0u` (all bits set) | Sentinel value meaning "no entity" |
| `DEFAULT_TICK_RATE` | `f32` | `60.0f` | Default fixed-update rate in Hz |
| `DEFAULT_WINDOW_WIDTH` | `i32` | `1280` | Default window width in pixels |
| `DEFAULT_WINDOW_HEIGHT` | `i32` | `720` | Default window height in pixels |

### HardwareTier (`core/types.h`, namespace `ffe`)

```cpp
enum class HardwareTier : u8 {
    RETRO    = 0,   // OpenGL 2.1, 512 MB VRAM, single-threaded
    LEGACY   = 1,   // OpenGL 3.3, 1 GB VRAM, single-threaded  <-- DEFAULT
    STANDARD = 2,   // OpenGL 4.5 / Vulkan, 2 GB VRAM, multi-threaded
    MODERN   = 3    // Vulkan, 4+ GB VRAM, multi-threaded + RT
};
```

**Tier helpers (constexpr, evaluated at compile time):**

| Function | Signature | Description |
|----------|-----------|-------------|
| `tierAtLeast` | `bool tierAtLeast(HardwareTier minimum)` | Returns true if compile-time tier >= `minimum` |
| `tierAtMost` | `bool tierAtMost(HardwareTier maximum)` | Returns true if compile-time tier <= `maximum` |
| `arenaDefaultSize` | `size_t arenaDefaultSize()` | Returns the default arena size for the current tier (512 KB to 16 MB) |

**Compile-time constant:**

- `CURRENT_TIER` -- the hardware tier set at CMake configure time via `FFE_TIER_VALUE`

### Result (`core/types.h`, namespace `ffe`)

A lightweight success/failure type. No heap allocation. No exceptions.

| Method | Signature | Description |
|--------|-----------|-------------|
| `ok` | `static Result ok()` | Create a success result |
| `fail` | `static Result fail(const char* msg)` | Create a failure result with a message (must be a string literal or static buffer) |
| `isOk` | `bool isOk() const` | Returns true if success |
| `operator bool` | `explicit operator bool() const` | Same as `isOk()` -- use in `if` statements |
| `message` | `const char* message() const` | Returns the error message (empty string on success) |

### ArenaAllocator (`core/arena_allocator.h`, namespace `ffe`)

A bump allocator for per-frame temporary data. Allocates from a single contiguous block. Reset is O(1). No individual frees. The Application resets it every frame automatically.

| Method | Signature | Description |
|--------|-----------|-------------|
| constructor | `explicit ArenaAllocator(size_t capacityBytes)` | Allocates the backing buffer (64-byte aligned). Logs FATAL and sets capacity to 0 on failure |
| `allocate` | `void* allocate(size_t size, size_t alignment = alignof(std::max_align_t))` | Allocate raw bytes. Returns `nullptr` if exhausted. Does NOT call constructors |
| `create<T>` | `template<typename T, typename... Args> T* create(Args&&... args)` | Allocate and construct a single `T`. Returns `nullptr` if exhausted |
| `allocateArray<T>` | `template<typename T> T* allocateArray(size_t count)` | Allocate and default-construct `count` elements of `T`. Returns `nullptr` if exhausted |
| `reset` | `void reset()` | Invalidate all previous allocations. O(1) -- just resets the offset |
| `capacity` | `size_t capacity() const` | Total bytes in the arena |
| `used` | `size_t used() const` | Bytes currently allocated |
| `remaining` | `size_t remaining() const` | Bytes available before exhaustion |

Non-copyable, non-movable.

### World / ECS (`core/ecs.h`, namespace `ffe`)

The Entity Component System. Built on top of EnTT. Entities are integer IDs. Components are plain structs. Systems are function pointers.

| Method | Signature | Description |
|--------|-----------|-------------|
| `createEntity` | `EntityId createEntity()` | Create a new entity. Returns its ID |
| `destroyEntity` | `void destroyEntity(EntityId id)` | Destroy an entity and all its components |
| `isValid` | `bool isValid(EntityId id) const` | Check if an entity ID is still alive |
| `addComponent<T>` | `template<typename T, typename... Args> T& addComponent(EntityId id, Args&&... args)` | Attach a component to an entity. Returns a reference to the new component |
| `removeComponent<T>` | `template<typename T> void removeComponent(EntityId id)` | Remove a component from an entity |
| `getComponent<T>` | `template<typename T> T& getComponent(EntityId id)` | Get a mutable reference to an entity's component. Undefined behaviour if the entity does not have it |
| `getComponent<T> const` | `template<typename T> const T& getComponent(EntityId id) const` | Get a const reference |
| `hasComponent<T>` | `template<typename T> bool hasComponent(EntityId id) const` | Check if an entity has a component |
| `view<Components...>` | `template<typename... Components> auto view()` | Get an iterable view of all entities that have ALL the listed components. Returns an EnTT view directly (no wrapper) |
| `registerSystem` | `void registerSystem(const SystemDescriptor& desc)` | Add a system to the world. Call before `sortSystems()` |
| `sortSystems` | `void sortSystems()` | Sort all registered systems by priority. Call once after all systems are registered |
| `systems` | `const std::vector<SystemDescriptor>& systems() const` | Get the sorted list of systems |
| `registry` | `entt::registry& registry()` | Escape hatch: direct access to the EnTT registry. Use only for advanced cases (context variables, etc.) |

### SystemDescriptor and SystemUpdateFn (`core/system.h`, namespace `ffe`)

```cpp
using SystemUpdateFn = void(*)(World& world, float dt);

struct SystemDescriptor {
    const char* name;           // For logging and profiling
    size_t nameLength;          // Cached strlen(name)
    SystemUpdateFn updateFn;    // The function to call each tick
    int32_t priority;           // Lower = runs first. Use multiples of 100
};
```

**Priority conventions:**

| Range | Purpose |
|-------|---------|
| 0-99 | Input polling |
| 100-199 | Gameplay / scripting |
| 200-299 | Physics |
| 300-399 | Animation |
| 400-499 | Audio |
| 500+ | Render preparation |

### Application (`core/application.h`, namespace `ffe`)

The top-level object. Creates the window, runs the main loop, owns all subsystems.

**ApplicationConfig struct:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `windowTitle` | `const char*` | `"FastFreeEngine"` | Window title bar text |
| `windowWidth` | `i32` | `1280` | Window width in pixels |
| `windowHeight` | `i32` | `720` | Window height in pixels |
| `tickRate` | `f32` | `60.0f` | Fixed-update frequency in Hz |
| `tier` | `HardwareTier` | `LEGACY` | Target hardware tier |
| `headless` | `bool` | `false` | If true, no window is created (for tests/CI) |

**Application class methods:**

| Method | Signature | Description |
|--------|-----------|-------------|
| constructor | `explicit Application(const ApplicationConfig& config)` | Set up config and allocate the frame arena |
| `run` | `int32_t run()` | Start the engine. Blocks until shutdown. Returns 0 on success, 1 on startup failure |
| `requestShutdown` | `void requestShutdown()` | Signal the engine to stop at the end of the current frame |
| `world` | `World& world()` | Access the ECS world (to create entities, add components, register systems) |
| `frameAllocator` | `ArenaAllocator& frameAllocator()` | Access the per-frame arena allocator |
| `config` | `const ApplicationConfig& config() const` | Read back the config |

Non-copyable, non-movable. There is exactly one Application.

**Main loop behaviour:** Fixed timestep with variable rendering. Systems run at `tickRate` Hz. Rendering happens as fast as possible (VSync on by default). The frame allocator is reset at the end of every frame.

### Logging (`core/logging.h`, namespace `ffe`)

**Functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `initLogging` | `void initLogging()` | Initialize the logging subsystem (called by Application) |
| `shutdownLogging` | `void shutdownLogging()` | Flush and shut down logging |
| `setLogLevel` | `void setLogLevel(LogLevel level)` | Set minimum severity. Messages below this are discarded |
| `setLogFile` | `void setLogFile(FILE* file)` | Redirect output. Pass `nullptr` for stdout. Caller owns the FILE* |
| `logMessage` | `void logMessage(LogLevel level, const char* system, const char* fmt, ...)` | Core log function. Do not call directly -- use the macros |

**LogLevel enum:**

`TRACE` = 0, `DEBUG` = 1, `INFO` = 2, `WARN` = 3, `ERR` = 4, `FATAL` = 5

Note: `ERR` not `ERROR` because `ERROR` is a Windows macro.

**Macros (use these, not `logMessage` directly):**

```cpp
FFE_LOG_TRACE(system, fmt, ...)   // Compiled out in Release builds
FFE_LOG_DEBUG(system, fmt, ...)   // Compiled out in Release builds
FFE_LOG_INFO(system, fmt, ...)
FFE_LOG_WARN(system, fmt, ...)
FFE_LOG_ERROR(system, fmt, ...)
FFE_LOG_FATAL(system, fmt, ...)   // Flushes output immediately
```

All macros take a `system` string (e.g. `"Core"`, `"Physics"`) as the first argument, then a `printf`-style format string and arguments.

**Output format:** `[HH:MM:SS.mmm] [LEVEL] [system] message`

## Common Usage Patterns

### 1. Minimal application setup

```cpp
#include "core/application.h"

int main() {
    ffe::ApplicationConfig config;
    config.windowTitle = "My First Game";
    config.windowWidth = 1280;
    config.windowHeight = 720;

    ffe::Application app(config);

    // Register your systems before run()
    // app.world().registerSystem(...);
    // app.world().sortSystems();  // Already called inside startup, but call again if you add more

    return app.run();
}
```

### 2. Creating entities with components

```cpp
void setupScene(ffe::World& world) {
    ffe::EntityId player = world.createEntity();
    world.addComponent<Transform>(player, glm::vec3{0.0f, 0.0f, 0.0f});
    world.addComponent<Sprite>(player);

    auto& sprite = world.getComponent<Sprite>(player);
    sprite.size = {64.0f, 64.0f};
    sprite.color = {1.0f, 0.0f, 0.0f, 1.0f}; // Red
}
```

### 3. Writing a system

```cpp
void movementSystem(ffe::World& world, float dt) {
    auto view = world.view<Transform, const Velocity>();
    for (auto entity : view) {
        auto& pos = view.get<Transform>(entity);
        const auto& vel = view.get<const Velocity>(entity);
        pos.position.x += vel.dx * dt;
        pos.position.y += vel.dy * dt;
    }
}

// Register it:
ffe::SystemDescriptor desc;
desc.name = "Movement";
desc.nameLength = 8;
desc.updateFn = movementSystem;
desc.priority = 100; // Gameplay range
world.registerSystem(desc);
world.sortSystems();
```

### 4. Using the arena allocator for temporary data

```cpp
void mySystem(ffe::World& world, float dt) {
    // Get the frame allocator from Application (you'll need to pass it or store a reference)
    // This memory is valid until the end of the current frame.
    auto* tempArray = app.frameAllocator().allocateArray<glm::vec2>(100);
    if (tempArray == nullptr) {
        FFE_LOG_ERROR("MySystem", "Frame arena exhausted!");
        return;
    }
    // Use tempArray... it will be automatically invalidated at frame end.
    // Do NOT store this pointer across frames.
}
```

### 5. Using Result for error handling

```cpp
ffe::Result loadConfig(const char* path) {
    FILE* f = fopen(path, "r");
    if (!f) {
        return ffe::Result::fail("Could not open config file");
    }
    // ... read config ...
    fclose(f);
    return ffe::Result::ok();
}

// Caller:
ffe::Result r = loadConfig("game.cfg");
if (!r) {
    FFE_LOG_ERROR("Config", "Load failed: %s", r.message());
}
```

## What Not To Do

### Do not store arena pointers across frames
```cpp
// BAD -- this pointer is invalidated when the frame allocator resets
glm::vec2* savedPtr = app.frameAllocator().allocateArray<glm::vec2>(10);
// ... next frame ...
savedPtr[0] = {1.0f, 2.0f}; // USE-AFTER-FREE -- undefined behaviour
```
The arena resets every frame. If you need data to survive across frames, allocate it on the heap or use a persistent allocator.

### Do not call getComponent without checking hasComponent first
```cpp
// BAD -- crashes if the entity doesn't have a Transform
auto& t = world.getComponent<Transform>(entity);

// GOOD
if (world.hasComponent<Transform>(entity)) {
    auto& t = world.getComponent<Transform>(entity);
}

// BEST -- use a view, which only gives you entities that have the components
auto view = world.view<Transform>();
for (auto entity : view) { /* safe */ }
```

### Do not use `new` or `malloc` in per-frame code
FFE forbids heap allocation in hot paths. Use the frame arena or pre-allocate at startup.

### Do not forget to call sortSystems after registering systems
Systems run in registration order until `sortSystems()` is called. If you skip it, your physics might run before input polling.

### Do not use exceptions for error handling
FFE disables exceptions in the engine core. Use `Result` or return values instead.

### Do not pass large structs by value
Pass by `const&` or move. This is enforced project-wide.

### Do not log with raw logMessage -- use the macros
The macros compile out TRACE and DEBUG in Release builds. Calling `logMessage` directly bypasses this.

## Tier Support

The core subsystem supports **all four tiers**: RETRO, LEGACY, STANDARD, and MODERN.

**Tier-specific behaviour:**

| Behaviour | RETRO | LEGACY | STANDARD | MODERN |
|-----------|-------|--------|----------|--------|
| Default arena size | 512 KB | 2 MB | 8 MB | 16 MB |
| Threading model | Single-core safe | Single-core safe | Multi-threaded | Multi-threaded |
| OpenGL version | 2.1 | 3.3 | 4.5 / Vulkan | Vulkan |

The `CURRENT_TIER` constant and `tierAtLeast()`/`tierAtMost()` helpers are resolved at compile time, so tier checks have zero runtime cost.

## Dependencies

| Dependency | What For |
|------------|----------|
| **EnTT** (header-only) | ECS implementation (entity storage, component pools, views) |
| **GLFW** | Window creation and input polling (used by Application) |
| **glad** | OpenGL function loading (used by Application startup) |
| **GLM** | Math types used in renderer components that Application references |
| **Tracy** | Frame profiling (ZoneScoped, FrameMark in the main loop) |
| **C++ standard library** | `<cstdio>`, `<cstdlib>`, `<chrono>`, `<mutex>`, `<vector>`, `<algorithm>` |

No other FFE subsystems depend on core. Core depends on nothing else in FFE -- it is the root of the dependency tree. The renderer depends on core, not the other way around (though `application.h` includes renderer headers because Application owns the rendering pipeline).
