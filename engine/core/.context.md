# engine/core — System Context

## System Purpose

The core subsystem is the foundation of FastFreeEngine. It provides the application lifecycle (window creation, main loop, shutdown), an Entity Component System (ECS) for organizing game objects, a per-frame arena allocator for fast temporary memory, a lightweight error type that replaces exceptions, structured logging, hardware tier detection, and fixed-width type aliases used everywhere in the engine. If you are making a game with FFE, every piece of your code depends on this subsystem.

---

### HardwareTier (`core/types.h`, namespace `ffe`)

```cpp
enum class HardwareTier : u8 {
    RETRO    = 0,   // OpenGL 2.1, 512 MB VRAM, single-threaded

```

### Result (`core/types.h`, namespace `ffe`)

| Method | Signature | Description |
|--------|-----------|-------------|
| `ok` | `static Result ok()` | Create a success result |
| `fail` | `static Result fail(const char* msg)` | Create a failure result with a message (must be a string literal or static buffer) |

### ArenaAllocator (`core/arena_allocator.h`, namespace `ffe`)

A bump allocator for per-frame temporary data. Allocates from a single contiguous block. Reset is O(1). No individual frees. The Application resets it every frame automatically.

| Method | Signature | Description |
|--------|-----------|-------------|
| constructor | `explicit ArenaAllocator(size_t capacityBytes)` | Allocates the backing buffer (64-byte aligned). Logs FATAL and sets capacity to 0 on failure |
| `allocate` | `void* allocate(size_t size, size_t alignment = alignof(std::max_align_t))` | Allocate raw bytes. Returns `nullptr` if exhausted. Does NOT call constructors |
| `create<T>` | `template<typename T, typename... Args> T* create(Args&&... args)` | Allocate and construct a single `T`. Returns `nullptr` if exhausted |
| `allocateArray<T>` | `template<typename T> T* allocateArray(size_t count)` | Allocate and default-construct `count` elements of `T`. Returns `nullptr` if exhausted |
| `reset` | `void reset()` | Invalidate all previous allocations. O(1) -- just resets the offset |
| `capacity` | `size_t capacity() const` | Total bytes in the arena |
| `used` | `size_t used() const` | Bytes currently allocated |
| `remaining` | `size_t remaining() const` | Bytes available before exhaustion |

Non-copyable, non-movable.

### World / ECS (`core/ecs.h`, namespace `ffe`)

The Entity Component System. Built on top of EnTT. Entities are integer IDs. Components are plain structs. Systems are function pointers.

| Method | Signature | Description |
|--------|-----------|-------------|
| `createEntity` | `EntityId createEntity()` | Create a new entity. Returns its ID |
| `destroyEntity` | `void destroyEntity(EntityId id)` | Destroy an entity and all its components |
| `clearAllEntities` | `void clearAllEntities()` | Destroy all entities and all components. Delegates to `entt::registry::clear()`. Does NOT clear ECS context values (ClearColor, CameraShake, etc.). Does NOT free heap memory owned by components -- the caller must handle that (e.g., `destroyTilemap()` on Tilemap components). This is a cold-path operation for scene transitions. |
| `isValid` | `bool isValid(EntityId id) const` | Check if an entity ID is still alive |
| `addComponent<T>` | `template<typename T, typename... Args> T& addComponent(EntityId id, Args&&... args)` | Attach a component to an entity. Returns a reference to the new component |
| `removeComponent<T>` | `template<typename T> void removeComponent(EntityId id)` | Remove a component from an entity |
| `getComponent<T>` | `template<typename T> T& getComponent(EntityId id)` | Get a mutable reference to an entity's component. Undefined behaviour if the entity does not have it |
| `getComponent<T> const` | `template<typename T> const T& getComponent(EntityId id) const` | Get a const reference |
| `hasComponent<T>` | `template<typename T> bool hasComponent(EntityId id) const` | Check if an entity has a component |
| `view<Components...>` | `template<typename... Components> auto view()` | Get an iterable view of all entities that have ALL the listed components. Returns an EnTT view directly (no wrapper) |
| `registerSystem` | `void registerSystem(const SystemDescriptor& desc)` | Add a system to the world. Call before `sortSystems()` |
| `sortSystems` | `void sortSystems()` | Sort all registered systems by priority. Call once after all systems are registered |
| `systems` | `const std::vector<SystemDescriptor>& systems() const` | Get the sorted list of systems |
| `registry` | `entt::registry& registry()` | Escape hatch: direct access to the EnTT registry. Use only for advanced cases (context variables, etc.) |

### SystemDescriptor and SystemUpdateFn (`core/system.h`, namespace `ffe`)

```cpp
using SystemUpdateFn = void(*)(World& world, float dt);

struct SystemDescriptor {
    const char* name;           // For logging and profiling
    size_t nameLength;          // Cached strlen(name)
    SystemUpdateFn updateFn;    // The function to call each tick
    int32_t priority;           // Lower = runs first. Use multiples of 100
};
```

**Priority conventions:**

| Range | Purpose |
|-------|---------|
| 0-99 | Input polling |
| 100-199 | Gameplay / scripting |
| 200-299 | Physics |
| 300-399 | Animation |
| 400-499 | Audio |
| 500+ | Render preparation |

**FFE_SYSTEM macro (`core/system.h`):**

```cpp
// Builds a SystemDescriptor from a string literal, function pointer, and priority.
// Avoids manual nameLength computation. Only pass string literals as name_str.
FFE_SYSTEM("MySystem", mySystemFn, 100)
```

Do NOT pass a runtime `const char*` variable as `name_str` — `sizeof()` returns the pointer size, not the string length. Use only string literals.

### ShutdownSignal (`core/ecs.h`, namespace `ffe`)

```cpp
struct ShutdownSignal {
    bool requested = false;
};
```

Stored in the ECS registry context by `Application::startup()`. Any system can request engine shutdown without holding an `Application*`:

```cpp
// Inside any system — shuts down the engine at the end of the current tick:
world.registry().ctx().get<ffe::ShutdownSignal>().requested = true;
```

`Application::run()` checks this after every tick and exits the loop if `requested` is `true`. There is no need to store or pass an `Application*` into systems.

### HudTextBuffer (`core/ecs.h`, namespace `ffe`)

```cpp
static constexpr u32 HUD_TEXT_BUFFER_SIZE = 256;

struct HudTextBuffer {
    char text[HUD_TEXT_BUFFER_SIZE] = "";
};
```

A 256-byte fixed char array stored in the ECS registry context. No heap allocation. Emplaced by `Application` during startup. Written by Lua scripts via `ffe.setHudText(text)` (truncated at 255 characters). Read by `EditorOverlay` each frame to render on-screen HUD text. Any system can also write to it directly:

```cpp
auto& hud = world.registry().ctx().get<ffe::HudTextBuffer>();
std::strncpy(hud.text, "Hello", ffe::HUD_TEXT_BUFFER_SIZE - 1);
hud.text[ffe::HUD_TEXT_BUFFER_SIZE - 1] = '\0';
```

Set `text[0] = '\0'` to clear the HUD.

### Application (`core/application.h`, namespace `ffe`)

The top-level object. Creates the window, runs the main loop, owns all subsystems.

**ApplicationConfig struct:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `windowTitle` | `const char*` | `"FastFreeEngine"` | Window title bar text |
| `windowWidth` | `i32` | `1280` | Window width in pixels |
| `windowHeight` | `i32` | `720` | Window height in pixels |
| `tickRate` | `f32` | `60.0f` | Fixed-update frequency in Hz |
| `tier` | `HardwareTier` | `LEGACY` | Target hardware tier |
| `headless` | `bool` | `false` | If true, no window is created (for tests/CI) |

**Application class methods:**

| Method | Signature | Description |
|--------|-----------|-------------|
| constructor | `explicit Application(const ApplicationConfig& config)` | Set up config and allocate the frame arena |
| `run` | `int32_t run()` | Start the engine. Blocks until shutdown. Returns 0 on success, 1 on startup failure |
| `requestShutdown` | `void requestShutdown()` | Signal the engine to stop at the end of the current frame |
| `world` | `World& world()` | Access the ECS world (to create entities, add components, register systems) |
| `frameAllocator` | `ArenaAllocator& frameAllocator()` | Access the per-frame arena allocator |
| `config` | `const ApplicationConfig& config() const` | Read back the config |

Non-copyable, non-movable. There is exactly one Application.

**Main loop behaviour:** Fixed timestep with variable rendering. Systems run at `tickRate` Hz. Rendering happens as fast as possible (VSync on by default). The frame allocator is reset at the end of every frame.

#### Editor-Hosted Mode

These methods allow an external application (the FFE Editor) to own the GLFW window and main loop while reusing the engine's subsystem initialisation and per-frame logic. The standalone `run()` path is unchanged; these are additive.

| Method | Signature | Description |
|--------|-----------|-------------|
| `setWindow` | `void setWindow(GLFWwindow* win)` | Hand an externally-created GLFW window to the engine. Must be called before `initSubsystems()`. The engine does NOT own or destroy this window |
| `window` | `GLFWwindow* window() const` | Returns the GLFW window pointer. The editor needs this for ImGui initialisation |
| `initSubsystems` | `bool initSubsystems()` | Initialise all subsystems (ECS, renderer, audio, physics, scripting) without creating a window or entering a main loop. Call `setWindow()` first. Returns false on failure |
| `shutdownSubsystems` | `void shutdownSubsystems()` | Clean shutdown of all subsystems in reverse init order. Call when the editor is closing |
| `tickOnce` | `void tickOnce(float dt)` | Run one fixed-timestep tick. `dt` is the time step in seconds (typically `1.0f / tickRate`) |
| `renderOnce` | `void renderOnce(float alpha)` | Render one frame. `alpha` is the interpolation factor [0, 1) for lerping between physics states to eliminate fixed-timestep jitter |

**Calling sequence for editor-hosted mode:**

1. Editor creates GLFW window and OpenGL context
2. `app.setWindow(window)`
3. `app.initSubsystems()`
4. Editor main loop: `app.tickOnce(dt)` then `app.renderOnce(alpha)` each frame
5. `app.shutdownSubsystems()` on exit

### Input System (`core/input.h`, namespace `ffe`)

The input system provides keyboard, mouse button, mouse position/delta, scroll delta, and action mapping. It is a thin stateful layer over GLFW callbacks. Call `updateInput()` once per tick before any systems run.

**Constants:**

| Name | Type | Value | Description |
|------|------|-------|-------------|
| `MAX_KEYS` | `i32` | 512 | Upper bound for the key state array |
| `MAX_MOUSE_BUTTONS` | `i32` | 5 | Number of tracked mouse buttons |
| `MAX_ACTIONS` | `i32` | 64 | Maximum registered actions |
| `MAX_BINDINGS_PER_ACTION` | `i32` | 4 | Maximum key/button bindings per action |

**Key enum (`ffe::Key`):** Strongly typed key codes matching GLFW values. Letter keys use bare letter names — `Key::W`, `Key::A`, `Key::S`, `Key::D`, `Key::ESCAPE` — **not** `Key::KEY_W` style. Full set: letters (A–Z as `Key::A`…`Key::Z`), number row (NUM_0–NUM_9), function keys (F1–F12), arrow keys (UP/DOWN/LEFT/RIGHT), modifiers (LEFT_SHIFT, RIGHT_SHIFT, LEFT_CTRL, RIGHT_CTRL, LEFT_ALT, RIGHT_ALT), common keys (SPACE, ESCAPE, ENTER, TAB, BACKSPACE). `Key::KEY_COUNT` = 512 is a sentinel — not a valid key.

**MouseButton enum (`ffe::MouseButton`):** `LEFT` = 0, `RIGHT` = 1, `MIDDLE` = 2, `EXTRA1` = 3, `EXTRA2` = 4.

**Lifecycle functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `initInput` | `void initInput(GLFWwindow* window)` | Initialize the input system. Call after GLFW window creation. Pass `nullptr` for headless mode |
| `shutdownInput` | `void shutdownInput()` | Clean up. Call before GLFW window destruction |
| `updateInput` | `void updateInput()` | Advance one tick: promote previous-frame state and process pending GLFW events. Call once at the top of each tick before any systems run |

**Keyboard query functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `isKeyPressed` | `bool isKeyPressed(Key key)` | True if the key went down THIS frame (was up last frame). Use for one-shot actions |
| `isKeyHeld` | `bool isKeyHeld(Key key)` | True if the key is down AND was also down last frame. Use for continuous movement |
| `isKeyReleased` | `bool isKeyReleased(Key key)` | True if the key went up THIS frame (was down last frame). Use for release events |
| `isKeyUp` | `bool isKeyUp(Key key)` | True if the key is currently up. Returns true most of the time — rarely the right choice |
| `isShiftDown` | `bool isShiftDown()` | True if LEFT_SHIFT or RIGHT_SHIFT is held |
| `isCtrlDown` | `bool isCtrlDown()` | True if LEFT_CTRL or RIGHT_CTRL is held |
| `isAltDown` | `bool isAltDown()` | True if LEFT_ALT or RIGHT_ALT is held |

Out-of-range key values (raw integer cast > MAX_KEYS) are bounds-checked and return `false`. This is safe but is a logic error — use the `Key` enum.

**Latched detection:** `isKeyPressed` and `isKeyReleased` use latched (edge-triggered) detection. If a key is pressed and released within the same tick, the press event is still reported as `true` on that tick — it is not missed. This behaviour is identical to `isMouseButtonPressed` / `isMouseButtonReleased`, which follow the same latched pattern. The latch is cleared at the next `updateInput()` call.

**Mouse query functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `mouseX` | `f64 mouseX()` | Current cursor X position in screen pixels (top-left origin) |
| `mouseY` | `f64 mouseY()` | Current cursor Y position in screen pixels (top-left origin) |
| `mouseDeltaX` | `f64 mouseDeltaX()` | X movement since last `updateInput()`. Zero if mouse did not move this tick |
| `mouseDeltaY` | `f64 mouseDeltaY()` | Y movement since last `updateInput()`. Zero if mouse did not move this tick |
| `scrollDeltaX` | `f64 scrollDeltaX()` | Horizontal scroll since last `updateInput()`. Zero if no scroll this tick |
| `scrollDeltaY` | `f64 scrollDeltaY()` | Vertical scroll since last `updateInput()`. Positive = scroll up |
| `isMouseButtonPressed` | `bool isMouseButtonPressed(MouseButton btn)` | True if the button went down this frame |
| `isMouseButtonHeld` | `bool isMouseButtonHeld(MouseButton btn)` | True if the button was held across frames |
| `isMouseButtonReleased` | `bool isMouseButtonReleased(MouseButton btn)` | True if the button went up this frame |
| `isMouseButtonUp` | `bool isMouseButtonUp(MouseButton btn)` | True if the button is currently up |
| `setCursorCaptured` | `void setCursorCaptured(bool captured)` | Lock/hide the cursor (for first-person look). When captured, `mouseDeltaX/Y` reports raw motion |
| `isCursorCaptured` | `bool isCursorCaptured()` | Returns current capture state |

**Action mapping functions:**

| Function | Signature | Description |
|----------|-----------|-------------|
| `registerAction` | `i32 registerAction(const char* name)` | Register a named action. Returns the action index (0–63), or -1 if `MAX_ACTIONS` is exceeded. Call at startup, not per-frame |
| `findAction` | `i32 findAction(const char* name)` | Look up an action by name. Returns -1 if not found. O(n) — call at init, cache the result |
| `bindActionKey` | `bool bindActionKey(i32 actionIndex, Key key)` | Bind a key to an action. Returns false if `actionIndex` is out of range or `MAX_BINDINGS_PER_ACTION` is exceeded |
| `bindActionMouseButton` | `bool bindActionMouseButton(i32 actionIndex, MouseButton btn)` | Bind a mouse button to an action. Same failure conditions as `bindActionKey` |
| `clearActionBindings` | `void clearActionBindings(i32 actionIndex)` | Remove all bindings for an action. Does nothing if `actionIndex` is out of range |
| `isActionPressed` | `bool isActionPressed(i32 actionIndex)` | True if ANY binding went down this frame |
| `isActionHeld` | `bool isActionHeld(i32 actionIndex)` | True if ANY binding is being held |
| `isActionReleased` | `bool isActionReleased(i32 actionIndex)` | True if ANY binding went up this frame |
| `isActionUp` | `bool isActionUp(i32 actionIndex)` | True if ALL bindings are currently up |

**IMPORTANT — `isActionUp` asymmetry:** `isActionPressed`, `isActionHeld`, and `isActionReleased` return true if **ANY** binding matches. `isActionUp` returns true only if **ALL** bindings are up. This is intentional (the action is only fully "at rest" when every bound key/button is up), but it can be surprising. If an action has both `Space` and `Gamepad_A` bound and the player holds both simultaneously, `isActionUp` returns false even after releasing Space.

## Common Usage Patterns

### 1. Minimal application setup

```cpp
#include "core/application.h"

int main() {
    ffe::ApplicationConfig config;
    config.windowTitle = "My First Game";
    config.windowWidth = 1280;
    config.windowHeight = 720;

    ffe::Application app(config);

    // Register your systems before run()
    // app.world().registerSystem(...);
    // app.world().sortSystems();  // Already called inside startup, but call again if you add more

    return app.run();
}
```

### 2. Creating entities with components

```cpp
void setupScene(ffe::World& world) {
    ffe::EntityId player = world.createEntity();
    world.addComponent<Transform>(player, glm::vec3{0.0f, 0.0f, 0.0f});
    world.addComponent<Sprite>(player);

    auto& sprite = world.getComponent<Sprite>(player);
    sprite.size = {64.0f, 64.0f};
    sprite.color = {1.0f, 0.0f, 0.0f, 1.0f}; // Red
}
```

### 3. Writing a system

```cpp
void movementSystem(ffe::World& world, float dt) {
    auto view = world.view<Transform, const Velocity>();
    for (auto entity : view) {
        auto& pos = view.get<Transform>(entity);
        const auto& vel = view.get<const Velocity>(entity);
        pos.position.x += vel.dx * dt;
        pos.position.y += vel.dy * dt;
    }
}

// Register it (use FFE_SYSTEM to avoid manual nameLength):
world.registerSystem(FFE_SYSTEM("Movement", movementSystem, 100));
world.sortSystems();
```

### 4. Using the arena allocator for temporary data

```cpp
void mySystem(ffe::World& world, float dt) {
    // Get the frame allocator from Application (you'll need to pass it or store a reference)
    // This memory is valid until the end of the current frame.
    auto* tempArray = app.frameAllocator().allocateArray<glm::vec2>(100);
    if (tempArray == nullptr) {
        FFE_LOG_ERROR("MySystem", "Frame arena exhausted!");
        return;
    }
    // Use tempArray... it will be automatically invalidated at frame end.
    // Do NOT store this pointer across frames.
}
```

### 5. Basic key and mouse button polling

```cpp
// Called from a system registered at priority 0-99 (input range)
void playerInputSystem(ffe::World& world, float dt) {
    // isKeyPressed — fires once on the frame the key goes down
    if (ffe::isKeyPressed(ffe::Key::SPACE)) {
        // Trigger a jump — happens exactly once per press
        startJump(world);
    }

    // isKeyHeld — continuous while the key is down
    const float speed = 200.0f;
    float dx = 0.0f;
    if (ffe::isKeyHeld(ffe::Key::D) || ffe::isKeyHeld(ffe::Key::RIGHT)) {
        dx += speed * dt;
    }
    if (ffe::isKeyHeld(ffe::Key::A) || ffe::isKeyHeld(ffe::Key::LEFT)) {
        dx -= speed * dt;
    }

    // isKeyReleased — fires once on the frame the key goes up
    if (ffe::isKeyReleased(ffe::Key::LEFT_SHIFT)) {
        stopSprint(world);
    }

    // Mouse buttons follow the same pattern
    if (ffe::isMouseButtonPressed(ffe::MouseButton::LEFT)) {
        fireWeapon(world);
    }
}
```

### 6. Action mapping setup and query

```cpp
// At startup — store the returned index, don't call findAction per frame
i32 g_actionJump   = -1;
i32 g_actionFire   = -1;
i32 g_actionSprint = -1;

void setupInputActions() {
    g_actionJump   = ffe::registerAction("Jump");
    g_actionFire   = ffe::registerAction("Fire");
    g_actionSprint = ffe::registerAction("Sprint");

    // Bind multiple keys to the same action — player can use either
    ffe::bindActionKey(g_actionJump, ffe::Key::SPACE);
    ffe::bindActionKey(g_actionJump, ffe::Key::W);

    ffe::bindActionMouseButton(g_actionFire, ffe::MouseButton::LEFT);
    ffe::bindActionKey(g_actionSprint, ffe::Key::LEFT_SHIFT);
}

// In your system:
void gameplaySystem(ffe::World& world, float dt) {
    if (ffe::isActionPressed(g_actionJump)) {
        startJump(world);   // Space OR W going down — either triggers it
    }
    if (ffe::isActionHeld(g_actionFire)) {
        holdTrigger(world);
    }
    // isActionUp is true only when ALL bindings are up simultaneously
    if (ffe::isActionUp(g_actionSprint)) {
        setWalking(world);
    }
}
```

### 7. Mouse movement and scroll (first-person look)

```cpp
void setupCamera() {
    ffe::setCursorCaptured(true);   // Hide and lock cursor
}

void cameraSystem(ffe::World& world, float dt) {
    // When cursor is captured, deltaX/Y reports raw relative motion
    const f64 dx = ffe::mouseDeltaX();
    const f64 dy = ffe::mouseDeltaY();

    const float sensitivity = 0.1f;
    rotateCameraYaw(world,   static_cast<float>(dx) * sensitivity);
    rotateCameraPitch(world, static_cast<float>(dy) * sensitivity);

    // Scroll for zoom or weapon cycling
    const f64 scrollY = ffe::scrollDeltaY(); // positive = scroll up
    if (scrollY != 0.0) {
        adjustZoom(world, static_cast<float>(scrollY));
    }

    // Toggle capture on Escape
    if (ffe::isKeyPressed(ffe::Key::ESCAPE)) {
        ffe::setCursorCaptured(!ffe::isCursorCaptured());
    }
}
```

### 8. Editor-hosted mode (embedding the engine)

```cpp
#include "core/application.h"

// The editor creates its own GLFW window and OpenGL context, then hands
// the window to the engine. The editor owns the main loop.
void editorInit(GLFWwindow* editorWindow) {
    ffe::ApplicationConfig config;
    config.windowTitle  = "FastFreeEngine Editor";
    config.windowWidth  = 1280;
    config.windowHeight = 720;
    config.tier         = ffe::HardwareTier::LEGACY;

    auto app = std::make_unique<ffe::Application>(config);
    app->setWindow(editorWindow);   // Hand over the window (do NOT call run())
    app->initSubsystems();          // Init ECS, renderer, audio, scripting
}

// Each frame in the editor loop:
void editorFrame(ffe::Application& app, float dt, float alpha) {
    app.tickOnce(dt);       // Run one fixed-timestep update
    app.renderOnce(alpha);  // Render one frame
    // Then render editor UI (ImGui) on top...
}

// On shutdown:
void editorShutdown(ffe::Application& app) {
    app.shutdownSubsystems();
}
```

### 9. Using Result for error handling

```cpp
ffe::Result loadConfig(const char* path) {
    FILE* f = fopen(path, "r");
    if (!f) {
        return ffe::Result::fail("Could not open config file");
    }
    // ... read config ...
    fclose(f);
    return ffe::Result::ok();
}

// Caller:
ffe::Result r = loadConfig("game.cfg");
if (!r) {
    FFE_LOG_ERROR("Config", "Load failed: %s", r.message());
}
```

## What Not To Do

### Do not store arena pointers across frames
```cpp
// BAD -- this pointer is invalidated when the frame allocator resets
glm::vec2* savedPtr = app.frameAllocator().allocateArray<glm::vec2>(10);
// ... next frame ...
savedPtr[0] = {1.0f, 2.0f}; // USE-AFTER-FREE -- undefined behaviour
```
The arena resets every frame. If you need data to survive across frames, allocate it on the heap or use a persistent allocator.

### Do not call getComponent without checking hasComponent first
```cpp
// BAD -- crashes if the entity doesn't have a Transform
auto& t = world.getComponent<Transform>(entity);

// GOOD
if (world.hasComponent<Transform>(entity)) {
    auto& t = world.getComponent<Transform>(entity);
}

// BEST -- use a view, which only gives you entities that have the components
auto view = world.view<Transform>();
for (auto entity : view) { /* safe */ }
```

### Do not use `new` or `malloc` in per-frame code
FFE forbids heap allocation in hot paths. Use the frame arena or pre-allocate at startup.

### Do not forget to call sortSystems after registering systems
Systems run in registration order until `sortSystems()` is called. If you skip it, your physics might run before input polling.

### Do not use exceptions for error handling
FFE disables exceptions in the engine core. Use `Result` or return values instead.

### Do not pass large structs by value
Pass by `const&` or move. This is enforced project-wide.

### Do not log with raw logMessage -- use the macros
The macros compile out TRACE and DEBUG in Release builds. Calling `logMessage` directly bypasses this.

### Do not call registerAction or findAction inside the game loop

```cpp
// BAD -- O(n) search every frame, and may fail once MAX_ACTIONS is hit
void mySystem(ffe::World& world, float dt) {
    i32 idx = ffe::findAction("Jump");  // Called 60 times per second
    if (ffe::isActionPressed(idx)) { ... }
}

// GOOD -- cache the index at startup
static i32 s_jumpAction = ffe::registerAction("Jump");

void mySystem(ffe::World& world, float dt) {
    if (ffe::isActionPressed(s_jumpAction)) { ... }
}
```

`registerAction` also has a hard limit of `MAX_ACTIONS` (64). Calling it in the loop will exhaust the table and return -1 repeatedly, then `isActionPressed(-1)` silently returns false.

### Do not call run() in editor-hosted mode

`run()` enters the engine's internal main loop and blocks until shutdown. In editor-hosted mode, the editor owns the main loop and calls `tickOnce()` / `renderOnce()` each frame instead.

```cpp
// BAD -- blocks forever, editor UI never renders
app->setWindow(editorWindow);
app->initSubsystems();
app->run();  // WRONG -- this takes over the main loop

// GOOD -- editor controls the loop
app->setWindow(editorWindow);
app->initSubsystems();
// ... in editor loop: app->tickOnce(dt); app->renderOnce(alpha);
```

### Do not forget setWindow() before initSubsystems()

In editor-hosted mode, the engine needs the GLFW window before initialising the renderer. Calling `initSubsystems()` without `setWindow()` will fail or create a second window.

### Do not compare mouse positions with ==

Mouse positions are `f64`. Floating-point exact equality is unreliable.

```cpp
// BAD
if (ffe::mouseX() == 640.0) { ... }

// GOOD -- check if mouse is within a region
if (ffe::mouseX() > 620.0 && ffe::mouseX() < 660.0) { ... }
```

### Do not misread the isActionUp asymmetry

`isActionPressed`, `isActionHeld`, and `isActionReleased` return true if **ANY** binding matches. `isActionUp` returns true only if **ALL** bindings are up.

```cpp
// Action "Jump" bound to: Space and W
// Player holds both Space AND W, then releases Space only.
ffe::isActionHeld(jump)    // true  — W is still held
ffe::isActionPressed(jump) // false — no new press this frame
ffe::isActionUp(jump)      // false — W is still held; ALL must be up
```

If you need "the action is no longer active", use `!isActionHeld(idx)` rather than `isActionUp(idx)` when multiple bindings are involved.

### Do not use raw integer casts to Key values outside the defined enum

The state array is bounds-checked and returns `false` for any value >= `MAX_KEYS`, so it will not crash. However, using raw integers is a logic error — not every integer in 0–511 is a named key, and GLFW does not guarantee those slots contain meaningful state.

```cpp
// BAD
bool pressed = ffe::isKeyPressed(static_cast<ffe::Key>(999)); // always false, logic error

// GOOD
bool pressed = ffe::isKeyPressed(ffe::Key::SPACE);
```

## Tier Support

The core subsystem supports **all four tiers**: RETRO, LEGACY, STANDARD, and MODERN.

**Tier-specific behaviour:**

| Behaviour | RETRO | LEGACY | STANDARD | MODERN |
|-----------|-------|--------|----------|--------|
| Default arena size | 512 KB | 2 MB | 8 MB | 16 MB |
| Threading model | Single-core safe | Single-core safe | Multi-threaded | Multi-threaded |
| OpenGL version | 2.1 | 3.3 | 4.5 / Vulkan | Vulkan |

The `CURRENT_TIER` constant and `tierAtLeast()`/`tierAtMost()` helpers are resolved at compile time, so tier checks have zero runtime cost.

**Input system tier support:** The input system is tier-agnostic — it operates identically on RETRO through MODERN. GLFW abstracts the OS input layer. Minimum tier is LEGACY (the FFE default), but there is no GPU dependency whatsoever. Input state arrays are stack/static — no heap allocation per frame.

### 8. Gamepad input polling

```cpp
void gamepadInputSystem(ffe::World& world, float dt) {
    // Check if a gamepad is connected
    if (!ffe::isGamepadConnected(0)) return;

    // D-pad or left stick for movement
    float dx = 0.0f, dy = 0.0f;

    if (ffe::isGamepadButtonHeld(0, ffe::GamepadButton::DPAD_LEFT)) {
        dx -= 200.0f * dt;
    }
    if (ffe::isGamepadButtonHeld(0, ffe::GamepadButton::DPAD_RIGHT)) {
        dx += 200.0f * dt;
    }

    // Analog stick (deadzone already applied by getGamepadAxis)
    const float stickX = ffe::getGamepadAxis(0, ffe::GamepadAxis::LEFT_X);
    dx += stickX * 200.0f * dt;

    // Face buttons for actions
    if (ffe::isGamepadButtonPressed(0, ffe::GamepadButton::A)) {
        startJump(world);
    }

    // Triggers (range: -1.0 released to 1.0 fully pressed)
    const float rightTrigger = ffe::getGamepadAxis(0, ffe::GamepadAxis::RIGHT_TRIGGER);
    if (rightTrigger > 0.0f) {
        accelerate(world, rightTrigger * dt);
    }
}
```

## Internal Utilities

### Platform Abstraction (`core/platform.h`, namespace `ffe`)

`platform.h` is a thin internal header that wraps OS-specific functions. **Game scripts do not call these functions directly — they are used internally by the engine's asset loading and path security code (texture loader, audio, mesh loader, scripting save/load).** Do not use them in game code.

#### `ffe::canonicalizePath`

```cpp
inline bool canonicalizePath(const char* path, char* outBuf, size_t outBufSize);
```

Cross-platform path canonicalization. Resolves `.` and `..` components and symlinks, returning the absolute canonical path in `outBuf`.

| Parameter | Description |
|-----------|-------------|
| `path` | Input path (absolute or relative) |
| `outBuf` | Output buffer to receive the canonical path |
| `outBufSize` | Size of `outBuf` in bytes |

Returns `true` and writes into `outBuf` on success. Returns `false` on failure.

**Platform semantic difference — important:**

| Platform | Underlying call | Existence requirement |
|----------|-----------------|-----------------------|
| Linux / macOS | `realpath()` | **Path must exist.** Returns false (sets `errno = ENOENT`) if the path or any component does not exist |
| Windows (MinGW / MSVC) | `_fullpath()` | **Path does not need to exist.** Performs lexical canonicalization only |

This means `canonicalizePath` may return `true` on Windows for a path that does not yet exist on disk. The engine's call sites always follow up with a `stat()` existence check, so this asymmetry is safe in practice. **Do not rely on a `true` return as an existence check — call `stat()` separately.**

**Buffer size note:** On Linux, the underlying `realpath()` does not accept a size argument — it uses `PATH_MAX` (4096) internally. The `outBufSize` parameter is therefore only enforced on Windows. All engine call sites pass a buffer of at least `PATH_MAX + 1` bytes to remain safe on both platforms. If you call this function directly, always allocate at least `PATH_MAX + 1` bytes for `outBuf`.

**Tier support:** All tiers. No GPU dependency. Load-time only.

## Prefab System (`core/prefab_system.h`, namespace `ffe`)

### System Purpose

The prefab system provides data-driven entity templates: define a single entity's components and their initial field values once in a JSON file, then instantiate that template as many times as needed at runtime with optional per-call field overrides. Without prefabs, spawning 200 trees means writing the same `addComponent` calls 200 times in Lua or C++. With prefabs, it is a single `loadPrefab` call at startup and a loop of `instantiatePrefab` calls thereafter. Prefabs are single-entity templates (contrast with `SceneSerialiser`, which loads whole multi-entity scenes). They are the primary tool for populating 3D scenes with repeated object types. M1 supports 3D components; 2D component support is deferred to M2.

---

### Public C++ API

#### `PrefabHandle` (`core/prefab_system.h`)

```cpp
struct PrefabHandle {
    uint32_t id = 0;           // 0 = null/invalid. Valid range: [1, 63].
    explicit operator bool() const { return id != 0; }
};

inline bool isValid(const PrefabHandle h) { return h.id != 0; }
```

Consistent opaque handle pattern used by `MeshHandle`, `TerrainHandle`, and all other FFE asset handles. A return value of `PrefabHandle{0}` from `loadPrefab` always indicates failure.

#### `PrefabOverrides` (`core/prefab_system.h`)

```cpp
struct PrefabOverrides {
    static constexpr int MAX = 8;   // Hard limit. First 8 overrides applied; extras dropped.

    // Append a float override. No-op (with warning) if count == MAX.
    void set(const char* component, const char* field, float v);

    // Append an int override. No-op (with warning) if count == MAX.
    void set(const char* component, const char* field, int v);

    // Append a bool override. No-op (with warning) if count == MAX.
    void set(const char* component, const char* field, bool v);
};
```

Entirely stack-allocated (~584 bytes). Pass by value to `instantiatePrefab` — no heap allocation. The `component` and `field` strings name the target (e.g. `"Transform3D"`, `"x"`); they are not the value. Component name and field name are truncated to 31 characters.

#### `PrefabSystem` (`core/prefab_system.h`)

| Method | Signature | Description |
|--------|-----------|-------------|
| `setAssetRoot` | `void setAssetRoot(std::string_view root)` | Set the asset root directory. `loadPrefab` rejects any path that resolves outside this directory after canonicalization. Call once before any `loadPrefab`. Default: process working directory. |
| `loadPrefab` | `PrefabHandle loadPrefab(std::string_view path)` | Load a prefab from a JSON file. Canonicalizes the path, enforces the asset root, rejects files larger than 1 MB and JSON nesting deeper than 8 levels, parses component data into a pool slot. Returns `PrefabHandle{0}` on any error; logs the reason. Cold path — call at startup or scene load. |
| `unloadPrefab` | `void unloadPrefab(PrefabHandle handle)` | Release a prefab pool slot. The handle is immediately invalid. Passing `PrefabHandle{0}` or an already-unloaded handle is a no-op. |
| `instantiatePrefab` | `EntityId instantiatePrefab(World& world, PrefabHandle handle, const PrefabOverrides& overrides = {})` | Create a new entity in `world`, apply the prefab's component template, then patch fields listed in `overrides`. Returns the new `EntityId`. Returns `NULL_ENTITY` if the handle is invalid or the slot is unoccupied. No heap allocation on this path. |
| `getPrefabCount` | `int getPrefabCount() const` | Return the number of currently loaded prefabs (0–63). |

**Pool capacity:** 64 slots (slot 0 reserved as null/invalid). Slots 1–63 are usable.

**Thread safety:** Single-threaded. All calls must be made from the main thread.

---

### Prefab JSON Format

A prefab file is a single JSON object. The `"components"` key is required. The `"name"` key is optional (used for logging; defaults to the slot index if omitted). All other top-level keys are ignored for forward compatibility.

```json
{
  "name": "tree",
  "components": {
    "Transform3D": {
      "x": 0.0,  "y": 0.0,  "z": 0.0,
      "rotX": 0.0, "rotY": 0.0, "rotZ": 0.0,
      "scaleX": 1.0, "scaleY": 1.0, "scaleZ": 1.0
    },
    "Mesh": {
      "path": "assets/models/tree.glb"
    },
    "Material3D": {
      "r": 0.4, "g": 0.7, "b": 0.3,
      "metallic": 0.0, "roughness": 0.8
    }
  }
}
```

All components are optional — a prefab can contain any subset. Unknown component keys are skipped with a log warning (forward compatibility). Unknown fields inside a known component are also silently ignored.

#### Supported component types (M1)

| Component key | Supported fields | Notes |
|---------------|-----------------|-------|
| `"Transform3D"` | `x, y, z, rotX, rotY, rotZ, scaleX, scaleY, scaleZ` | All float. Defaults: position (0,0,0), rotation (0,0,0), scale (1,1,1). Rotation values are Euler degrees, XYZ order. |
| `"Mesh"` | `path` | String path to a `.glb` asset. The mesh must already be loaded via `ffe.loadMesh()` before instantiation — the prefab system does not load GPU resources. In M1, there is no path-to-handle lookup in the mesh loader, so the Mesh component will be added with an invalid handle; load your mesh separately and set the handle yourself. |
| `"Material3D"` | `r, g, b, metallic, roughness` | All float. Defaults: color (1,1,1), metallic 0.0, roughness 0.5. |
| `"PBRMaterial"` | `albedoR, albedoG, albedoB, metallic, roughness, ao` | All float. Defaults: albedo (1,1,1), metallic 0.0, roughness 0.5, ao 1.0. |

#### Constraints

| Constraint | Value |
|------------|-------|
| Maximum file size | 1 MB |
| Maximum JSON nesting depth | 8 levels |
| Maximum pool capacity | 63 prefabs simultaneously (slots 1–63) |
| Maximum overrides per `instantiatePrefab` call | 8 |

### Common Usage Patterns

#### 1. Load a prefab at startup and instantiate it at a position

```lua
-- At scene load — cold path, call once
local treeHandle = ffe.loadPrefab("assets/prefabs/tree.json")
if treeHandle == 0 then
    -- loadPrefab logs the error; 0 means failure
    return
end

-- Instantiate one tree at (10, 0, 5)
local eid = ffe.instantiatePrefab(treeHandle, {
    Transform3D = { x = 10.0, y = 0.0, z = 5.0 }
})
```

#### 2. Instantiate many copies with varying positions

```lua
-- Scatter 20 trees along the X axis
for i = 1, 20 do
    ffe.instantiatePrefab(treeHandle, {
        Transform3D = { x = i * 3.0, y = 0.0, z = 0.0 }
    })
end
```

#### 3. Override multiple components at once

```lua
-- Instantiate with custom position, scale, and PBR albedo
local eid = ffe.instantiatePrefab(treeHandle, {
    Transform3D = {
        x = -5.0, y = 0.0, z = 10.0,
        scaleX = 2.0, scaleY = 2.0, scaleZ = 2.0
    },
    PBRMaterial = {
        albedoR = 0.2, albedoG = 0.6, albedoB = 0.1
    }
})
```

#### 4. Unload a prefab when no longer needed

```lua
-- Free the pool slot (e.g., on level transition)
ffe.unloadPrefab(treeHandle)
-- treeHandle is now invalid; do not use it after this point
```

#### C++ usage

```cpp
ffe::PrefabSystem prefabs;
prefabs.setAssetRoot("assets/");

ffe::PrefabHandle tree = prefabs.loadPrefab("assets/prefabs/tree.json");
if (!tree) { /* handle error */ return; }

// No overrides — use template values as-is
ffe::EntityId e1 = prefabs.instantiatePrefab(world, tree);

// With overrides — stack-allocated, no heap
ffe::PrefabOverrides ov;
ov.set("Transform3D", "x", 10.0f);
ov.set("Transform3D", "z",  5.0f);
ffe::EntityId e2 = prefabs.instantiatePrefab(world, tree, ov);

prefabs.unloadPrefab(tree);
```

### What NOT To Do

**Do not call `ffe.loadPrefab` inside the game loop.**
`loadPrefab` opens a file, calls `stat`, reads and parses JSON. It is a cold-path operation. Call it once at startup or scene load, store the handle, and reuse it.

```lua
-- BAD — file I/O every frame
function update(dt)
    local h = ffe.loadPrefab("assets/prefabs/tree.json")  -- file read each tick
    ffe.instantiatePrefab(h, { ... })
end

-- GOOD — load once, instantiate many times
local treeHandle = ffe.loadPrefab("assets/prefabs/tree.json")
function update(dt)
    if shouldSpawnTree then
        ffe.instantiatePrefab(treeHandle, { ... })
    end
end
```

**Do not pass more than 8 overrides per `instantiatePrefab` call.**
The `PrefabOverrides` struct holds a maximum of 8 entries. The 9th and subsequent calls to `overrides.set()` (C++) or the 9th field pair in the Lua table are silently dropped with a log warning. If you need more than 8 field overrides, create a more specific prefab file instead.

**Do not pass string values as override field values in Lua.**
Only numbers and booleans are accepted. String values (e.g., a mesh path override) are ignored with a warning. String-field overrides are an M2 feature.

```lua
-- BAD — string value is ignored, warning logged
ffe.instantiatePrefab(handle, { Mesh = { path = "other_model.glb" } })

-- GOOD — load a separate prefab file for each mesh variant
local oakHandle = ffe.loadPrefab("assets/prefabs/oak_tree.json")
local pineHandle = ffe.loadPrefab("assets/prefabs/pine_tree.json")
```

**Do not use a handle after calling `unloadPrefab`.**
The pool slot is freed immediately. Any subsequent `instantiatePrefab` call with the old handle returns `NULL_ENTITY` (C++) or `ffe.NULL_ENTITY` (Lua) and logs an error.

**Do not attempt path traversal in `loadPrefab`.**
Paths are canonicalized and verified against the declared asset root before any file is opened. `ffe.loadPrefab("../../etc/passwd")` returns 0, logs an error, and opens nothing.

**Do not rely on `rotX`/`rotY`/`rotZ` single-axis overrides composing additively.**
Due to quaternion storage in `Transform3D`, a single-axis rotation override rebuilds the full quaternion using identity (0) for the other two axes. A `rotX = 45` override sets rotation to 45° around X with 0° on Y and Z — it does not add 45° to whatever rotation the template already specifies. To set a specific full orientation, override all three axes explicitly, or set the transform after instantiation.
