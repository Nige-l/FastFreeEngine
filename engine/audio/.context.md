# engine/audio -- Audio Subsystem

AI-native context file. Written by api-designer after implementation review.

---

## System Purpose

The audio subsystem provides fire-and-forget sound effect playback and looping music streaming for FFE games. It wraps **miniaudio** (a public-domain single-header C library) for device output and **stb_vorbis** (another single-header library) for OGG decoding. WAV files are decoded by miniaudio directly.

The design follows the same global-module-singleton pattern as `ffe::rhi`: one audio device, one mixer, one output stream. Sound effects are decoded fully into float32 PCM at load time and held in a fixed-capacity table of 256 slots. Up to 32 voices can play simultaneously. Music tracks stream from disk via stb_vorbis per-callback decode -- one active music track at a time.

The audio callback runs on a background thread managed by miniaudio. Game code (main thread) communicates with the callback via a lock-free SPSC command ring buffer -- the main thread never blocks waiting for the audio thread, and the audio thread never stalls the game loop.

**Supported formats:** WAV (RIFF magic) and OGG Vorbis (OggS magic). Format is detected by magic bytes, not file extension.

**Supported tier:** LEGACY and above. RETRO is not supported.

## Public API

Header: `engine/audio/audio.h`
Namespace: `ffe::audio`

All functions must be called from the main thread unless noted otherwise.

### Handle Type

```cpp
struct SoundHandle {
    u32 id = 0;  // 0 is invalid; non-zero is valid
};

inline bool isValid(SoundHandle h);
```

`SoundHandle` is an opaque value. `id == 0` means invalid (load failed or handle was never set). Use `isValid()` to check before passing to any playback function.

### Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_AUDIO_VOICES` | 32 | Maximum simultaneously active voices |
| `MAX_AUDIO_PATH` | 4096 | Maximum length for audio asset paths |
| `AUDIO_MAX_DECODED_BYTES` | 10 MB | Maximum decoded PCM size per sound (rejects sounds longer than ~30 seconds at 44.1 kHz stereo) |
| `AUDIO_MAX_FILE_BYTES` | 32 MB | Maximum raw file size accepted before decoding begins |

### Lifecycle

| Function | Signature | Description |
|----------|-----------|-------------|
| `init` | `bool init(bool headless = false)` | Initialise the audio subsystem. Call once from the main thread before any other audio function. Creates the miniaudio device (44.1 kHz, float32 stereo) and starts the audio callback thread. Returns false if the device could not be opened -- the engine continues without audio and all playback functions become no-ops. Calling `init()` a second time logs a warning and returns false. When `headless = true`, skips device creation entirely (safe for CI and tests without audio hardware); `loadSound()` and `unloadSound()` still work for path-validation testing; `playSound()` becomes a no-op; `isAudioAvailable()` returns false. |
| `shutdown` | `void shutdown()` | Stop the audio thread, free all loaded sounds, close the device. Call once at shutdown, after all other audio calls. Safe to call if `init()` was never called (no-op in that case). Must be called from the main thread with no concurrent audio calls in flight. |

### Loading and Unloading

| Function | Signature | Description |
|----------|-----------|-------------|
| `loadSound` | `SoundHandle loadSound(const char* path, const char* assetRoot)` | Load a WAV or OGG audio file and return a handle to the decoded PCM buffer. `path` must be a relative path with no traversal sequences (see Security section). `assetRoot` must be an absolute path to an existing directory. Path safety is checked first, before any syscall. Returns `SoundHandle{0}` on failure and logs the error. Performs `stat()`, `realpath()`, `fopen()`, and full PCM decode -- call this at load time only, not per-frame. Subject to `AUDIO_MAX_DECODED_BYTES` limit (10 MB). For music tracks that exceed this limit, use `loadMusic()` instead. |
| `loadMusic` | `SoundHandle loadMusic(const char* path, const char* assetRoot)` | Lightweight music loader for streaming playback via `playMusic()`. Validates the path and stores the canonical path but does NOT decode the file to PCM. The `AUDIO_MAX_DECODED_BYTES` limit does not apply (only `AUDIO_MAX_FILE_BYTES` applies). The returned handle can be passed to `playMusic()` but NOT to `playSound()` (no decoded PCM is available). Same path validation and security checks as `loadSound()`. |
| `unloadSound` | `void unloadSound(SoundHandle handle)` | Free the PCM buffer for a loaded sound. If the sound is currently playing, the active voice is stopped atomically (under mutex) before freeing -- no use-after-free window. No-op for `SoundHandle{0}`. After this call, the handle must not be used. Works for handles from both `loadSound()` and `loadMusic()`. Call from the main thread only. |

### Playback — Sound Effects

| Function | Signature | Description |
|----------|-----------|-------------|
| `playSound` | `void playSound(SoundHandle handle, float volume = 1.0f)` | Play a one-shot sound effect. Fire-and-forget -- there is no way to stop an individual voice once started. No-op if the handle is invalid, the audio system is not initialised, or the system is in headless mode. If all `MAX_AUDIO_VOICES` slots are occupied, the sound is dropped and a warning is logged (not an error). `volume`: per-instance volume multiplier `[0.0, 1.0]`, clamped. NaN and Inf are treated as 0.0. May be called from the main thread. Do NOT call from within the audio callback. |

### Playback — Music

Music uses streaming decode (stb_vorbis per-callback). One track at a time. Music volume is independent of the master (sound effects) volume.

| Function | Signature | Description |
|----------|-----------|-------------|
| `playMusic` | `void playMusic(SoundHandle handle, bool loop = true)` | Start playing a music track loaded by `loadSound()` or `loadMusic()`. If a track is already playing, it is stopped first. `loop = true` plays indefinitely; `loop = false` plays once then stops. No-op if handle is invalid, audio is unavailable, or in headless mode. The handle is used to retrieve the validated file path -- the file is re-opened for streaming. |
| `stopMusic` | `void stopMusic()` | Stop the current music track immediately. No-op if no music is playing. |
| `setMusicVolume` | `void setMusicVolume(float volume)` | Set music volume independently of the master volume. Clamped to `[0.0, 1.0]`. NaN/Inf treated as 0.0. |
| `getMusicVolume` | `float getMusicVolume()` | Query current music volume. Returns 1.0 if audio is not initialised. Reads an atomic -- safe to call per-frame. |
| `isMusicPlaying` | `bool isMusicPlaying()` | Returns true if a music track is currently active. Reads an atomic -- safe to call per-frame. |

### Volume Control

| Function | Signature | Description |
|----------|-----------|-------------|
| `setMasterVolume` | `void setMasterVolume(float volume)` | Set the master volume applied to all audio output. Clamped to `[0.0, 1.0]`. Values outside this range are clamped and a warning is logged. NaN and Inf are treated as 0.0. No-op if the audio system is not initialised. |
| `getMasterVolume` | `float getMasterVolume()` | Query the current master volume `[0.0, 1.0]`. Returns 1.0 if the audio system is not initialised. |

### Diagnostics

| Function | Signature | Description |
|----------|-----------|-------------|
| `isAudioAvailable` | `bool isAudioAvailable()` | Returns true if the audio device was successfully opened. Returns false if `init()` failed, was called with `headless = true`, or was never called. Use this to conditionally show or disable audio settings UI. |
| `getActiveVoiceCount` | `u32 getActiveVoiceCount()` | Returns the number of sound voices currently active `[0, MAX_AUDIO_VOICES]`. Reads an atomic counter -- no mutex, safe to call per-frame. Useful for debugging and audio budget monitoring. Returns 0 if the audio system is not initialised. |

## Common Usage Patterns

### 1. Initialise, load a sound, play it, and clean up at shutdown

```cpp
#include "audio/audio.h"

// In Application::startup() or equivalent:
const bool audioOk = ffe::audio::init(false);
if (!audioOk) {
    // Audio is unavailable (no device, or device failed to open).
    // The engine continues without audio -- all playback calls become no-ops.
    FFE_LOG_WARN("Game", "Audio unavailable -- continuing without sound");
}

// Load sound effects at scene load time, not per-frame.
// path is relative; assetRoot is an absolute directory.
const char* assetRoot = "/home/user/mygame/assets/audio";
ffe::audio::SoundHandle jumpSfx = ffe::audio::loadSound("sfx/jump.wav", assetRoot);
ffe::audio::SoundHandle coinSfx = ffe::audio::loadSound("sfx/coin.ogg", assetRoot);

// Check that the sounds loaded before using them.
if (!ffe::audio::isValid(jumpSfx)) {
    FFE_LOG_WARN("Game", "jump.wav failed to load -- jump sound will be silent");
}

// Play sounds in response to game events -- not per-frame:
void onPlayerJump() {
    ffe::audio::playSound(jumpSfx);           // full volume (default 1.0)
}

void onCoinCollected() {
    ffe::audio::playSound(coinSfx, 0.7f);     // 70% volume
}

// At application shutdown -- after all audio calls:
ffe::audio::unloadSound(jumpSfx);
ffe::audio::unloadSound(coinSfx);
ffe::audio::shutdown();
```

### 2. Adjust master volume at runtime (for audio settings UI)

```cpp
// Set volume from a slider value in [0.0, 1.0].
// Out-of-range values are clamped; NaN/Inf are treated as 0.0.
ffe::audio::setMasterVolume(0.5f);   // 50% volume

// Read the current volume for display in a settings panel.
const float current = ffe::audio::getMasterVolume();  // 0.5f
```

### 3. Headless / CI usage (no audio hardware)

```cpp
// In test code or CI environments without a sound card:
ffe::audio::init(true);  // headless = true: no device opened

// loadSound() and unloadSound() still work for path-validation testing.
ffe::audio::SoundHandle h = ffe::audio::loadSound("sfx/test.wav", assetRoot);
// ... validate the handle, check for SoundHandle{0} on failure ...
ffe::audio::unloadSound(h);

// playSound() is a no-op in headless mode.
// isAudioAvailable() returns false.
assert(!ffe::audio::isAudioAvailable());

ffe::audio::shutdown();
```

### 4. Integration inside an ECS system -- play on event boundaries, not per-frame

```cpp
// Correct usage inside an ECS system:
// playSound() is called only when a discrete game event fires,
// not on every tick. This satisfies the "no per-frame file I/O" rule
// (playSound itself does no I/O -- but you must still avoid calling
// loadSound inside a system update).

void playerJumpSystem(ffe::World& world, float dt) {
    // isActionPressed returns true on exactly one frame when the action fires.
    if (ffe::isActionPressed(g_actionJump)) {
        // One call per jump event, not 60 calls per second.
        ffe::audio::playSound(g_jumpSfx);
    }
}
```

### 5. Check the voice budget during development

```cpp
// After a busy combat scene, verify you are not saturating the voice budget.
// If this fires, reduce the number of simultaneous sound effects.
const u32 voices = ffe::audio::getActiveVoiceCount();
if (voices >= ffe::audio::MAX_AUDIO_VOICES) {
    FFE_LOG_WARN("Audio", "Voice budget full: %u/%u voices active -- sounds may be dropped",
                 voices, ffe::audio::MAX_AUDIO_VOICES);
}
```

### 6. Play a looping background music track

```cpp
// Load the music track at scene start (same as sound effects).
// It can be WAV or OGG -- OGG is preferred for long music tracks (smaller files).
const char* assetRoot = "/home/user/mygame/assets/audio";
ffe::audio::SoundHandle bgMusic = ffe::audio::loadSound("music/theme.ogg", assetRoot);

if (!ffe::audio::isValid(bgMusic)) {
    FFE_LOG_WARN("Game", "theme.ogg failed to load -- no background music");
}

// Start looping (default: loop = true).
ffe::audio::playMusic(bgMusic);

// Set music volume independent of sound effects volume.
ffe::audio::setMusicVolume(0.6f);  // music at 60%

// On scene transition -- stop music before loading the next scene.
ffe::audio::stopMusic();

// At shutdown -- unload after stopping.
ffe::audio::stopMusic();
ffe::audio::unloadSound(bgMusic);
ffe::audio::shutdown();
```

### 7. Switch music track on scene transition

```cpp
// Load tracks for both scenes at startup.
ffe::audio::SoundHandle overworldMusic = ffe::audio::loadSound("music/overworld.ogg", assetRoot);
ffe::audio::SoundHandle dungeonMusic   = ffe::audio::loadSound("music/dungeon.ogg",   assetRoot);

// When entering the dungeon:
ffe::audio::playMusic(dungeonMusic);  // automatically stops overworld music

// When leaving the dungeon:
ffe::audio::playMusic(overworldMusic); // automatically stops dungeon music

// For a one-shot jingle that plays once and then stops:
ffe::audio::playMusic(victoryJingle, false /* loop = false */);

// Check if the jingle has finished:
if (!ffe::audio::isMusicPlaying()) {
    // Resume normal background music
    ffe::audio::playMusic(overworldMusic);
}
```

## What NOT to Do

### Do not call loadSound() per frame

`loadSound()` performs `stat()`, `realpath()`, `fopen()`, full PCM decode, and a heap allocation for the decoded PCM buffer. Calling it per frame is a severe performance violation that will cause drops on any hardware tier.

```cpp
// BAD -- per-frame loadSound is a critical performance violation
void gameSystem(ffe::World& world, float dt) {
    auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);  // WRONG: every frame
    ffe::audio::playSound(h);
    // Also leaks: the handle is never unloaded
}

// GOOD -- load once at scene start, keep the handle, play many times
// At scene load:
g_jumpSfx = ffe::audio::loadSound("sfx/jump.wav", assetRoot);

// In response to the game event (not per-frame):
ffe::audio::playSound(g_jumpSfx);
```

### Do not use a SoundHandle after unloadSound()

`unloadSound()` frees the decoded PCM buffer. Any voice using that buffer is stopped under the mutex before the free -- there is no use-after-free. But after the call, the handle's slot is recycled and will be reused by a future `loadSound()`. Always clear the handle variable after unloading.

```cpp
// BAD -- use after unload
ffe::audio::unloadSound(jumpSfx);
ffe::audio::playSound(jumpSfx);   // undefined behaviour -- handle slot was recycled

// GOOD -- null the handle after unloading
ffe::audio::unloadSound(jumpSfx);
jumpSfx = ffe::audio::SoundHandle{};   // id = 0; isValid() now returns false
```

### Do not call audio functions before init()

All audio functions check `s_state.initialised` internally and are silent no-ops if `init()` has not been called. However, silently doing nothing when you expect sound is a logic error. Always call `init()` first.

```cpp
// BAD -- no init before use
auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);  // returns SoundHandle{0}
ffe::audio::playSound(h);   // no-op (not initialised) -- silent failure

// GOOD
ffe::audio::init();
auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);
ffe::audio::playSound(h);
```

### Do not call audio functions from the audio callback thread

`playSound()` posts to an SPSC ring buffer consumed by the audio callback. Calling `playSound()` from within the callback would write from the consumer side, violating the SPSC contract and producing a data race. Game code should never run inside the miniaudio callback.

### Do not pass absolute paths or traversal sequences to loadSound()

`isAudioPathSafe()` is the first operation in `loadSound()`, before any syscall. Paths starting with `/` or `\`, containing `../`, or containing drive letters (`C:`) are rejected immediately.

```cpp
// BAD -- rejected before any filesystem access
auto h = ffe::audio::loadSound("/etc/passwd", assetRoot);         // SoundHandle{0}
auto h = ffe::audio::loadSound("../../../etc/passwd", assetRoot); // SoundHandle{0}

// GOOD -- relative path within the asset root
auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);
```

### Do not call playMusic() with a handle that was already unloaded

`playMusic()` retrieves the canonical file path from the sound table. If the handle was unloaded via `unloadSound()`, the table slot is cleared. The PLAY_MUSIC command will be silently dropped in the callback with an error log.

```cpp
// BAD -- use after unload
ffe::audio::unloadSound(bgMusic);
ffe::audio::playMusic(bgMusic);  // error logged, no music plays

// GOOD -- only unload music handles when you are truly done with them
ffe::audio::stopMusic();
ffe::audio::unloadSound(bgMusic);
bgMusic = ffe::audio::SoundHandle{};
```

### Do not use playMusic() for short one-shot sound effects

`playMusic()` opens a stb_vorbis streaming state -- there is ~10 µs overhead per `playMusic()` call as the decoder opens the file and reads the OGG headers. For short sound effects that trigger many times per second, use `playSound()` instead.

### Do not ignore the return value of loadSound()

`loadSound()` can fail for many reasons: file not found, unsupported format, decoded size exceeds the 10 MB limit, sound table full (max 256 sounds). Always check `isValid()`. The specific error reason is in the log.

```cpp
// BAD -- silently uses an invalid handle
auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);
ffe::audio::playSound(h);   // silent no-op if load failed

// GOOD -- check and handle failure explicitly
auto h = ffe::audio::loadSound("sfx/jump.wav", assetRoot);
if (!ffe::audio::isValid(h)) {
    FFE_LOG_WARN("Game", "jump.wav failed to load -- jump will be silent");
}
ffe::audio::playSound(h);   // still safe (no-op on invalid handle), but now the bug is visible
```

## Security

The audio subsystem enforces the following constraints on file loading. All of these run before the decoder is invoked:

1. **Path safety check first (SEC-1):** `isAudioPathSafe()` is the first operation in `loadSound()`. Rejects null, empty, absolute paths (Unix and Windows), drive letters, UNC paths, and all traversal sequences (`../`, `..\`, `/..`, `\..`). Uses `strnlen` bounded at `MAX_AUDIO_PATH` -- not `strlen`.
2. **realpath() canonicalisation (SEC-1):** After string validation, `realpath()` resolves symlinks and any encoding tricks the string check missed. The canonical path is then verified to start with the `assetRoot` prefix. If it does not, the load is rejected.
3. **File size limit before decoding (SEC-2):** `stat()` is called before `fopen()`. Files larger than `AUDIO_MAX_FILE_BYTES` (32 MB) are rejected before the decoder touches them.
4. **Format allowlist by magic bytes (SEC-3):** Only WAV (RIFF magic: `52 49 46 46`) and OGG (OggS magic: `4F 67 67 53`) are accepted. The first 4 bytes of the file are read and checked before the decoder is invoked.
5. **Decoder output validation before cast (HIGH-2 / SEC-5):** `frameCount`, `channels`, and `sampleRate` from the decoder are validated as positive and sane before any cast to `u64`. This prevents a negative `int` from the decoder being silently cast to `0xFFFFFFFFFFFFFFFF` in the subsequent size multiplication.
6. **Decoded size limit in u64 arithmetic (SEC-4):** Decoded PCM size is computed in `u64` arithmetic after validation and compared to `AUDIO_MAX_DECODED_BYTES` (10 MB).
7. **Volume clamping (SEC-6):** All volume inputs pass through `clampVolume()`. NaN and Inf are replaced with 0.0.
8. **Atomic voice/buffer teardown (LOW-4):** `unloadSound()` deactivates any active voice AND frees the PCM buffer in the same mutex critical section -- no window exists between deactivation and free.

## Tier Support

| Tier | Supported | Notes |
|------|-----------|-------|
| RETRO | **No** | Not supported. RETRO (~2005) hardware may lack a reliable audio API compatible with miniaudio. Games targeting RETRO have no audio. This is an explicit design decision, not an oversight. |
| LEGACY | **Yes** (primary target) | 44.1 kHz float32 stereo output; WAV and OGG formats; up to 32 simultaneous SFX voices; music streaming via stb_vorbis (one track at a time); no per-frame allocations; lock-free SPSC command ring. miniaudio's ALSA/PulseAudio/CoreAudio/WASAPI backends are stable on 2012-era hardware. |
| STANDARD | **Yes** | Identical behaviour to LEGACY. |
| MODERN | **Yes** | Identical behaviour to LEGACY. |

The audio callback runs on its own thread managed by miniaudio. The game loop's performance impact is: one lock-free ring-buffer write per `playSound()` / `playMusic()` call (on game-event boundaries, not per-frame), plus a single mutex acquisition per callback that covers both SFX mixing and music decode. This is well within LEGACY-tier budget.

## Dependencies

| Dependency | Source | Role |
|------------|--------|------|
| **miniaudio v0.11.25** | `third_party/miniaudio.h` (embedded, public domain) | Platform audio device abstraction; WAV decoding; audio callback thread management |
| **stb_vorbis v1.22** | `third_party/stb_vorbis.c` (embedded, public domain) | OGG Vorbis decoding |
| `ffe::core` types | `engine/core/types.h` | `u8`, `u32`, `u64` primitive types |
| `ffe::core` logging | `engine/core/logging.h` | `FFE_LOG_ERROR`, `FFE_LOG_WARN`, `FFE_LOG_INFO` macros |

Neither `miniaudio` nor `stb_vorbis` is in `vcpkg.json`. They are embedded single-header libraries following the same pattern as `stb_image` and `glad`.

The audio subsystem has no dependency on: renderer, ECS, physics, input, or scripting.
