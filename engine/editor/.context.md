# engine/editor — System Context

## System Purpose

The editor overlay is a debug-only ImGui interface compiled behind `#ifdef FFE_EDITOR` (Debug builds). It provides a real-time performance panel (FPS, frame time, entity count, audio voice count) and an entity inspector that lets developers select entities and edit their Transform components at runtime. This is an internal development tool, not a user-facing API.

---

## Public API

### EditorOverlay (`editor/editor.h`, namespace `ffe::editor`)

All code is guarded by `#ifdef FFE_EDITOR`. The class does not exist in Release builds.

| Method | Signature | Description |
|--------|-----------|-------------|
| `init` | `void init(GLFWwindow* window)` | Initialize ImGui context, install GLFW+OpenGL3 backends. Safe to call multiple times (no-ops after first). Must be called after GLFW window creation |
| `shutdown` | `void shutdown()` | Tear down ImGui backends and destroy the context. Safe to call if not initialised |
| `beginFrame` | `void beginFrame()` | Start a new ImGui frame. Call once per frame before `render()`. No-ops if not initialised |
| `render` | `void render(World& world)` | Draw all overlay panels (if visible) and submit ImGui draw data to OpenGL. Call once per frame after `beginFrame()` and after all game rendering |
| `toggle` | `void toggle()` | Toggle overlay visibility on/off. Typically bound to F1 |
| `isVisible` | `bool isVisible() const` | Returns `true` if the overlay is currently shown |
| `wantsMouse` | `bool wantsMouse() const` | Returns `true` if ImGui wants to capture mouse input (e.g., cursor is over an ImGui window). Use this to suppress game input when the overlay is active |
| `wantsKeyboard` | `bool wantsKeyboard() const` | Returns `true` if ImGui wants to capture keyboard input (e.g., a text field is focused). Use this to suppress game input when the overlay is active |

### Panels (private, for reference)

| Panel | Description |
|-------|-------------|
| Performance | FPS (1-second rolling average), frame time in ms, live entity count, active audio voice count |
| Entity Inspector | Two-column layout: scrollable entity list (left) with editable Transform fields and read-only Sprite info (right) |

## Common Usage Patterns

### 1. Integration in Application (init / beginFrame / render / shutdown cycle)

```cpp
#ifdef FFE_EDITOR
    ffe::editor::EditorOverlay m_editor;
#endif

// During startup, after GLFW window creation:
#ifdef FFE_EDITOR
    m_editor.init(window);
#endif

// In the main loop, once per frame:
#ifdef FFE_EDITOR
    m_editor.beginFrame();
#endif

// ... game systems tick, game rendering happens ...

// After game rendering, before glfwSwapBuffers:
#ifdef FFE_EDITOR
    m_editor.render(world);
#endif

// During shutdown:
#ifdef FFE_EDITOR
    m_editor.shutdown();
#endif
```

### 2. Toggle with F1

```cpp
if (ffe::isKeyPressed(ffe::Key::F1)) {
#ifdef FFE_EDITOR
    m_editor.toggle();
#endif
}
```

### 3. Suppressing game input when overlay is active

```cpp
#ifdef FFE_EDITOR
    const bool editorWantsMouse = m_editor.wantsMouse();
    const bool editorWantsKeyboard = m_editor.wantsKeyboard();
#else
    constexpr bool editorWantsMouse = false;
    constexpr bool editorWantsKeyboard = false;
#endif

if (!editorWantsMouse && ffe::isMouseButtonPressed(ffe::MouseButton::LEFT)) {
    // Handle game click
}
if (!editorWantsKeyboard && ffe::isKeyPressed(ffe::Key::SPACE)) {
    // Handle game action
}
```

## What Not To Do

### Do not call beginFrame or render without calling init first

Both methods silently no-op if the overlay is not initialised, but relying on this is a logic error. Always call `init()` during startup.

### Do not use EditorOverlay in headless mode

The overlay requires a live GLFW window and an OpenGL 3.3 context. Passing `nullptr` to `init()` or calling it without a valid GL context will crash ImGui's backend initialisation. In headless builds (`ApplicationConfig::headless = true`), do not instantiate or call the overlay.

### Do not call render before beginFrame in a given frame

ImGui requires `NewFrame()` before `Render()`. Calling `render()` without a preceding `beginFrame()` in the same frame is undefined behaviour in ImGui.

### Do not forget to guard all usage with `#ifdef FFE_EDITOR`

The `EditorOverlay` class does not exist in Release builds. Any unguarded reference will cause a compilation error.

### Do not render game content after the overlay render call

The overlay must be the last thing drawn before the buffer swap so that it appears on top of all game content.

## Tier Support

| Tier | Supported | Notes |
|------|-----------|-------|
| RETRO | No | Requires OpenGL 3.3 (ImGui backend uses `#version 330`) |
| LEGACY | Yes | Minimum supported tier |
| STANDARD | Yes | |
| MODERN | Yes | |

The overlay has negligible GPU cost -- a few textured quads for the UI panels. It does not affect the 60 fps budget on any supported tier.

## Dependencies

| Dependency | What For |
|------------|----------|
| **Dear ImGui** | All UI rendering (windows, text, drag floats, selectables) |
| **imgui_impl_glfw** | GLFW input backend with callback chaining |
| **imgui_impl_opengl3** | OpenGL 3.3 rendering backend |
| **ffe_core** | `World`, `Transform`, `Sprite`, `EntityId`, `u32` types, EnTT registry access |
| **ffe_renderer** | `RenderSystem` header (included for component types) |
| **ffe_audio** | `ffe::audio::getActiveVoiceCount()` for the performance panel |
| **EnTT** | Direct registry access for entity counting (`storage->size() - storage->free_list()`) |
