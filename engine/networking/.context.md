# engine/networking -- System Context

## System Purpose

The networking subsystem provides a complete client-server multiplayer stack for FFE. It is built in three layers: (1) a binary packet format with bounds-checked serialisation (`PacketReader`/`PacketWriter`), (2) an ENet-based transport layer with per-connection rate limiting (`ServerTransport`/`ClientTransport`), and (3) a high-level game networking layer with automatic ECS replication, snapshot interpolation, and game messaging (`NetworkServer`/`NetworkClient`/`network_system`). The system follows a "server XOR client" model -- only one mode is active at a time. All external input (network packets) is treated as untrusted and validated before processing. No virtual functions, no `std::function`, and no per-frame heap allocations in hot paths. This subsystem has no GPU involvement and runs on all tiers from LEGACY upward.

### Layer 1: Packet Format (`networking/packet.h`)

#### PacketType Enum

Wire-format packet type identifiers. Values are fixed -- do not reorder or reassign.

| Value | Name | Description |
|-------|------|-------------|
| `0x01` | `CONNECT_REQUEST` | Client requests connection |
| `0x02` | `CONNECT_ACCEPT` | Server accepts connection |
| `0x03` | `CONNECT_REJECT` | Server rejects connection |
| `0x04` | `DISCONNECT` | Graceful disconnect |
| `0x10` | `INPUT` | Client input payload |
| `0x11` | `SNAPSHOT_FULL` | Full game state snapshot |
| `0x12` | `SNAPSHOT_DELTA` | Delta-compressed state update |
| `0x20` | `EVENT` | Game event |
| `0x21` | `RPC` | Remote procedure call |
| `0xF0` | `HEARTBEAT` | Keep-alive ping |

| Function | Description |
|----------|-------------|
| `bool isValidPacketType(uint8_t t)` | Returns `true` if `t` maps to a known `PacketType` value. Use to validate incoming data before casting. |

#### PacketHeader Struct

On-wire header -- 6 bytes total.

| Field | Type | Description |
|-------|------|-------------|
| `type` | `PacketType` | Packet type identifier |
| `channel` | `uint8_t` | ENet channel (0 = reliable, 1 = unreliable-sequenced, 2 = reliable-ordered) |
| `sequence` | `uint16_t` | Sequence number for ordering/dedup |
| `payloadLength` | `uint16_t` | Length of the payload following the header |

#### Size Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_PACKET_SIZE` | `1200` | Maximum total packet size in bytes (MTU-safe) |
| `HEADER_SIZE` | `6` | Size of `PacketHeader` on the wire |
| `MAX_PAYLOAD_SIZE` | `1194` | Maximum payload bytes (`MAX_PACKET_SIZE - HEADER_SIZE`) |

#### Header Helpers

| Function | Description |
|----------|-------------|
| `bool writeHeader(PacketWriter& writer, const PacketHeader& header)` | Serialise a `PacketHeader` into the writer. Returns `false` if insufficient space. |
| `bool readHeader(PacketReader& reader, PacketHeader& header)` | Deserialise a `PacketHeader` from the reader. Returns `false` on bounds error. |

#### PacketWriter

Bounds-checked binary serialisation into a caller-owned buffer. Every write returns `false` and sets an internal error flag if the buffer would overflow.

| Method | Description |
|--------|-------------|
| `PacketWriter(uint8_t* buffer, uint16_t capacity)` | Construct over a pre-allocated buffer. |
| `bool writeU8(uint8_t val)` | Write 1 byte. |
| `bool writeU16(uint16_t val)` | Write 2 bytes (little-endian). |
| `bool writeU32(uint32_t val)` | Write 4 bytes (little-endian). |
| `bool writeF32(float val)` | Write 4-byte float. |
| `bool writeBytes(const uint8_t* data, uint16_t count)` | Write a raw byte span. |
| `bool writeString(const char* str)` | Write a length-prefixed string (u16 length prefix). |
| `uint16_t bytesWritten() const` | Number of bytes written so far. |
| `bool hasError() const` | `true` if any write failed (overflow). |
| `const uint8_t* data() const` | Pointer to the start of the buffer. |

#### PacketReader

Bounds-checked binary deserialisation from a non-owning data pointer. Every read returns `false` and sets an internal error flag on underflow. Once an error occurs, all subsequent reads also return `false`.

| Method | Description |
|--------|-------------|
| `PacketReader(const uint8_t* data, uint16_t size)` | Construct over received data. |
| `bool readU8(uint8_t& out)` | Read 1 byte. |
| `bool readU16(uint16_t& out)` | Read 2 bytes (little-endian). |
| `bool readU32(uint32_t& out)` | Read 4 bytes (little-endian). |
| `bool readF32(float& out)` | Read 4-byte float. **Rejects NaN and Inf values.** |
| `bool readBytes(uint8_t* out, uint16_t count)` | Read a raw byte span. |
| `bool readString(char* out, uint16_t maxLen)` | Read a length-prefixed string (u16 prefix). Fails if the encoded length exceeds `maxLen`. |
| `uint16_t remaining() const` | Bytes remaining in the buffer. |
| `bool hasError() const` | `true` if any read failed. |

### Layer 2: Transport (`networking/transport.h`, `networking/connection.h`)

#### Global Init / Shutdown

| Function | Description |
|----------|-------------|
| `bool initNetworking()` | Initialise ENet. Call once at application startup before creating any transport. Returns `false` on failure. |
| `void shutdownNetworking()` | Tear down ENet. Call once at application shutdown after all transports are destroyed. |

#### ConnectionState Enum (`networking/connection.h`)

| Value | Description |
|-------|-------------|
| `DISCONNECTED` | No active connection |
| `CONNECTING` | Connection handshake in progress |
| `CONNECTED` | Fully connected |
| `DISCONNECTING` | Graceful disconnect in progress |

#### ConnectionId Struct (`networking/connection.h`)

Lightweight value type identifying a remote peer.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | `uint32_t` | `0xFFFFFFFF` | Peer index. `0xFFFFFFFF` means invalid. |

| Function | Description |
|----------|-------------|
| `bool isValid(ConnectionId c)` | Returns `true` if `c.id != 0xFFFFFFFF`. |

#### ConnectionRateLimit Struct (`networking/connection.h`)

Per-connection rate limiter enforced server-side. Packets exceeding the limits are silently dropped.

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_PACKETS_PER_SECOND` | `100` | Maximum inbound packets per connection per second |
| `MAX_BYTES_PER_SECOND` | `65536` | Maximum inbound bytes per connection per second (64 KB/s) |

| Method | Description |
|--------|-------------|
| `bool allowPacket(uint32_t packetSize, float currentTime)` | Returns `true` if the packet is within rate limits. Automatically rolls over the 1-second window. |

#### ReceivedPacket Struct (`networking/connection.h`)

Non-owning view of an inbound packet, passed to receive callbacks. The `data` pointer is valid only for the duration of the callback.

| Field | Type | Description |
|-------|------|-------------|
| `sender` | `ConnectionId` | Which peer sent this packet |
| `channel` | `uint8_t` | ENet channel it arrived on |
| `data` | `const uint8_t*` | Pointer to raw packet bytes (non-owning) |
| `dataLength` | `uint16_t` | Length of `data` in bytes |

#### TransportConfig Struct (`networking/transport.h`)

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `port` | `uint16_t` | `7777` | UDP port to bind/connect |
| `maxClients` | `uint32_t` | `32` | Maximum simultaneous clients (server only) |
| `channelCount` | `uint32_t` | `3` | Number of ENet channels |
| `incomingBandwidth` | `uint32_t` | `0` | Incoming bandwidth limit in bytes/sec (`0` = unlimited) |
| `outgoingBandwidth` | `uint32_t` | `0` | Outgoing bandwidth limit in bytes/sec (`0` = unlimited) |

#### ServerTransport (`networking/transport.h`)

Non-copyable, non-movable server that listens for client connections. Uses raw function pointer callbacks (no `std::function`).

| Method | Description |
|--------|-------------|
| `ServerTransport()` | Default constructor. |
| `~ServerTransport()` | Destructor -- calls `stop()` if running. |
| `bool start(const TransportConfig& config)` | Bind to `config.port` and begin accepting connections. Returns `false` on failure. |
| `void stop()` | Disconnect all clients and release the ENet host. |
| `bool isRunning() const` | `true` if the server is actively listening. |
| `void poll(uint32_t timeoutMs = 0)` | Process inbound events and invoke callbacks. Call once per frame. `timeoutMs = 0` means non-blocking. |
| `bool send(ConnectionId client, uint8_t channel, const uint8_t* data, uint16_t len, bool reliable)` | Send a packet to a specific client. Returns `false` if the client is invalid. |
| `void broadcast(uint8_t channel, const uint8_t* data, uint16_t len, bool reliable)` | Send a packet to all connected clients. |
| `void disconnect(ConnectionId client)` | Gracefully disconnect a specific client. |
| `uint32_t clientCount() const` | Number of currently connected clients. |

**Server Callbacks (set before calling `start`):**

| Setter | Signature | Description |
|--------|-----------|-------------|
| `setConnectCallback(cb, userData)` | `void(*)(ConnectionId, void*)` | Called when a client connects. |
| `setDisconnectCallback(cb, userData)` | `void(*)(ConnectionId, void*)` | Called when a client disconnects. |
| `setReceiveCallback(cb, userData)` | `void(*)(const ReceivedPacket&, void*)` | Called for each received packet. |

#### ClientTransport (`networking/transport.h`)

| Method | Description |
|--------|-------------|

| `~ClientTransport()` | Destructor -- calls `disconnect()` if connected. |
| `bool connect(const char* host, uint16_t port, uint32_t channelCount = 3)` | Initiate connection to the given host and port. Returns `false` on immediate failure. |
| `void disconnect()` | Gracefully disconnect from the server. |
| `bool isConnected() const` | `true` if currently connected. |
| `void poll(uint32_t timeoutMs = 0)` | Process inbound events and invoke callbacks. Call once per frame. |
| `bool send(uint8_t channel, const uint8_t* data, uint16_t len, bool reliable)` | Send a packet to the server. Returns `false` if not connected. |

**Client Callbacks (set before calling `connect`):**

| Setter | Signature | Description |
|--------|-----------|-------------|
| `setConnectCallback(cb, userData)` | `void(*)(void*)` | Called when connection to the server is established. |
| `setDisconnectCallback(cb, userData)` | `void(*)(void*)` | Called when disconnected from the server. |

### Layer 3: Replication (`networking/replication.h`)

#### Function Pointer Types

| Type | Signature | Description |
|------|-----------|-------------|
| `SerializeFn` | `uint16_t(*)(const void*, uint8_t*, uint16_t)` | Serialize a component into a buffer. Returns bytes written, or 0 on failure. |
| `DeserializeFn` | `bool(*)(void*, const uint8_t*, uint16_t)` | Deserialize a component from a buffer. Returns true on success. |
| `InterpolateFn` | `void(*)(void*, const void*, const void*, float)` | Interpolate between two component states. May be null (snap to newer). |

#### ReplicatedComponent Struct

| Field | Type | Description |
|-------|------|-------------|
| `componentId` | `uint16_t` | Unique ID for wire protocol |
| `maxSerializedSize` | `uint16_t` | Max bytes when serialised |
| `serialize` | `SerializeFn` | Serialization function |
| `deserialize` | `DeserializeFn` | Deserialization function |
| `interpolate` | `InterpolateFn` | Interpolation function (may be nullptr for snap) |

#### ReplicationRegistry

Tracks which component types replicate over the wire. Fixed-size array, no heap allocation.

| Method | Description |
|--------|-------------|
| `bool registerComponent(uint16_t componentId, uint16_t maxSize, SerializeFn ser, DeserializeFn deser, InterpolateFn interp = nullptr)` | Register a component type. Returns false if full (max 32) or ID already registered. |
| `const ReplicatedComponent* find(uint16_t componentId) const` | Look up by wire ID. Returns nullptr if not found. |
| `uint16_t count() const` | Number of registered component types. |
| `const ReplicatedComponent* components() const` | Direct array access (0..count()-1). |

#### Replication Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_REPLICATED_COMPONENTS` | `32` | Maximum registered component types |
| `MAX_SNAPSHOT_ENTITIES` | `256` | Maximum entities per snapshot |
| `MAX_ENTITY_COMPONENT_DATA` | `256` | Maximum serialised bytes per entity |
| `SNAPSHOT_BUFFER_SIZE` | `16` | Ring buffer capacity for client interpolation |
| `COMPONENT_ID_TRANSFORM` | `1` | Built-in wire ID for Transform |
| `COMPONENT_ID_TRANSFORM3D` | `2` | Built-in wire ID for Transform3D |
| `TRANSFORM_SERIALIZED_SIZE` | `28` | 7 floats (pos.xyz, rot, scale.xyz) |
| `TRANSFORM3D_SERIALIZED_SIZE` | `40` | 10 floats (pos.xyz, quat.xyzw, scale.xyz) |

#### Snapshot Structures

**EntitySnapshot** -- serialised state of one entity:

| Field | Type | Description |
|-------|------|-------------|
| `entityId` | `uint32_t` | Entity identifier |
| `dataSize` | `uint16_t` | Bytes used in `componentData` |
| `componentMask` | `uint16_t` | Bitmask of which components are present |
| `componentData` | `uint8_t[256]` | Serialised component data |

**Snapshot** -- full game state at one tick:

| Field | Type | Description |
|-------|------|-------------|
| `tick` | `uint32_t` | Network tick number |
| `entityCount` | `uint32_t` | Number of entities in this snapshot |
| `entities` | `EntitySnapshot[256]` | Entity data array |

#### SnapshotBuffer

Ring buffer of recent snapshots for client-side interpolation. Heap-allocated internally (~1 MB).

| Method | Description |
|--------|-------------|
| `SnapshotBuffer()` | Constructor -- allocates snapshot array. |
| `void push(const Snapshot& snapshot)` | Push a snapshot (overwrites oldest if full). |
| `const Snapshot* latest() const` | Most recently pushed snapshot, or nullptr if empty. |
| `bool getInterpolationPair(uint32_t renderTick, const Snapshot*& a, const Snapshot*& b, float& t) const` | Find two snapshots bracketing `renderTick` for interpolation. Sets `a` (older), `b` (newer), `t` (factor in [0,1]). Returns false if fewer than 2 snapshots stored. |
| `uint32_t count() const` | Number of snapshots currently stored (0..16). |

#### Default Component Registration

| Function | Description |
|----------|-------------|
| `void registerDefaultComponents(ReplicationRegistry& registry)` | Register Transform (id=1) and Transform3D (id=2) with serialize/deserialize/interpolate. |
| `uint16_t serializeTransform(const void*, uint8_t*, uint16_t)` | Serialize Transform to 28 bytes. |
| `bool deserializeTransform(void*, const uint8_t*, uint16_t)` | Deserialize Transform from 28 bytes. |
| `void interpolateTransform(void*, const void*, const void*, float)` | Linearly interpolate Transform. |
| `uint16_t serializeTransform3D(const void*, uint8_t*, uint16_t)` | Serialize Transform3D to 40 bytes. |
| `bool deserializeTransform3D(void*, const uint8_t*, uint16_t)` | Deserialize Transform3D from 40 bytes. |
| `void interpolateTransform3D(void*, const void*, const void*, float)` | Interpolate Transform3D (slerp for quaternion rotation). |

### Prediction (`networking/prediction.h`)

Client-side prediction with server reconciliation. Records local input commands, applies them immediately for responsive gameplay, and corrects when the server's authoritative state diverges. Works without networking for local testing.

No virtual functions, no `std::function`, no per-frame heap allocations. All buffers are fixed-size arrays.

#### InputCommand Struct

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `tickNumber` | `uint32_t` | `0` | Client tick when input was recorded (engine-managed) |
| `inputBits` | `uint32_t` | `0` | Bitfield of pressed inputs (up to 32 buttons) |
| `dt` | `float` | `0.0f` | Delta time for this tick (engine-managed) |
| `aimX` | `float` | `0.0f` | Optional aim direction X |
| `aimY` | `float` | `0.0f` | Optional aim direction Y |

#### Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_PREDICTION_BUFFER` | `64` | Maximum number of input commands stored in the circular buffer |

#### Function Pointer Types

| Type | Signature | Description |
|------|-----------|-------------|
| `MoveFn` | `void(*)(World&, uint32_t entityId, const InputCommand&, void* userData)` | Applies one InputCommand to an entity's position. No virtual dispatch. |
| `InputCallbackFn` | `void(*)(uint32_t clientId, const InputCommand&, void* userData)` | Server-side callback when a client input packet arrives. |

#### PredictionBuffer

Circular buffer of `InputCommand` structs. Fixed-size array (`MAX_PREDICTION_BUFFER = 64`), no heap allocation.

| Method | Description |
|--------|-------------|
| `void record(const InputCommand& cmd)` | Store an input command. Overwrites oldest if full. |
| `const InputCommand* get(uint32_t tickNumber) const` | Retrieve a command by tick number. Returns nullptr if not found. |
| `void discardBefore(uint32_t tickNumber)` | Discard all inputs before the given tick. |
| `uint32_t count() const` | Number of commands currently stored. |
| `void clear()` | Clear all stored commands. |

#### ClientPrediction

Records input, applies predicted movement locally, and reconciles with server state.

| Method | Description |
|--------|-------------|
| `void setMovementFunction(MoveFn fn, void* userData)` | Set the function used to apply inputs to entities. |
| `void setLocalEntity(uint32_t entityId)` | Set the entity that is locally predicted. |
| `uint32_t localEntity() const` | Get the local entity ID. |
| `void recordAndPredict(World& world, const InputCommand& cmd)` | Called each client tick: records input and applies predicted movement. Increments the tick, stores the command, and calls the movement function. |
| `float reconcile(World& world, uint32_t serverTick, float serverX, float serverY, float serverZ)` | Called when a server snapshot arrives. If error exceeds threshold (0.1 units), snaps entity to server position and replays all buffered inputs after `serverTick`. Returns the reconciliation error magnitude. |
| `uint32_t currentTick() const` | Current client prediction tick. |
| `float lastError() const` | Last reconciliation error magnitude. |

### Server Input Handling

| Lua Function | Parameters | Returns | Description |
|-------------|------------|---------|-------------|
| `ffe.onServerInput(callback)` | `callback`: function(clientId, inputTable) or nil | nil | Register/clear a callback for when a client sends input. The `inputTable` contains: `{bits, aimX, aimY, tick, dt}`. Server-side only. |

## Common Usage Patterns

### 1. Lua: Starting a Multiplayer Server

```lua
function ffe.onInit()

    if not ok then
        print("Failed to start server!")

    end)

    ffe.onNetworkMessage(function(senderId, msgType, data)
        if msgType == 1 then  -- e.g., chat message

    end)
end
```

### 4. C++: Registering Custom Components for Replication

```cpp
#include "networking/replication.h"
#include "networking/network_system.h"
using namespace ffe::networking;

// Custom component
struct Health { float current; float max; };
constexpr uint16_t COMPONENT_ID_HEALTH = 10;
constexpr uint16_t HEALTH_SERIALIZED_SIZE = 8; // 2 floats

uint16_t serializeHealth(const void* comp, uint8_t* buf, uint16_t maxSize) {
    if (maxSize < HEALTH_SERIALIZED_SIZE) return 0;
    const auto* h = static_cast<const Health*>(comp);
    std::memcpy(buf, &h->current, 4);
    std::memcpy(buf + 4, &h->max, 4);
    return HEALTH_SERIALIZED_SIZE;
}

bool deserializeHealth(void* comp, const uint8_t* buf, uint16_t size) {
    if (size < HEALTH_SERIALIZED_SIZE) return false;
    auto* h = static_cast<Health*>(comp);
    std::memcpy(&h->current, buf, 4);
    std::memcpy(&h->max, buf + 4, 4);
    return true;
}

// Register before starting server/client
auto& reg = getReplicationRegistry();
reg.registerComponent(COMPONENT_ID_HEALTH, HEALTH_SERIALIZED_SIZE,
                      serializeHealth, deserializeHealth, nullptr);
```

### 5. Lua: Client-Side Prediction (Sending Input)

```lua
function ffe.onInit()
    ffe.onConnected(function()
        -- Create local player entity
        local id = ffe.createEntity()
        ffe.addTransform(id, 0, 0, 0, 1, 1)
        ffe.addPreviousTransform(id)
        ffe.addSprite(id, 1, 40, 40, 0.3, 0.7, 1.0, 1.0, 1)

        -- Tell the prediction system which entity is ours
        ffe.setLocalPlayer(id)
    end)

    ffe.connectToServer("127.0.0.1", 7777)
end

local INPUT_UP    = 1
local INPUT_DOWN  = 2
local INPUT_LEFT  = 4
local INPUT_RIGHT = 8
local SPEED       = 250.0

function update(entityId, dt)
    -- Build input bits from keyboard state
    local bits = 0
    if ffe.isKeyHeld(ffe.KEY_W) then bits = bits + INPUT_UP end
    if ffe.isKeyHeld(ffe.KEY_S) then bits = bits + INPUT_DOWN end
    if ffe.isKeyHeld(ffe.KEY_A) then bits = bits + INPUT_LEFT end
    if ffe.isKeyHeld(ffe.KEY_D) then bits = bits + INPUT_RIGHT end

    -- Send to server (engine records for reconciliation)
    ffe.sendInput({bits = bits, aimX = 0, aimY = 0})

    -- Apply predicted movement locally for responsive feel
    -- (The server will reconcile if the result diverges)
    if bits ~= 0 and localEntity then
        local buf = {}
        if ffe.fillTransform(localEntity, buf) then
            local dx, dy = 0, 0
            if bits >= INPUT_RIGHT then dx = dx + SPEED * dt end
            if bits >= INPUT_LEFT  then dx = dx - SPEED * dt end
            if bits >= INPUT_UP    then dy = dy + SPEED * dt end
            if bits >= INPUT_DOWN  then dy = dy - SPEED * dt end
            ffe.setTransform(localEntity, buf.x + dx, buf.y + dy,
                             buf.rotation, buf.scaleX, buf.scaleY)
        end
    end

    -- Monitor prediction quality
    local err = ffe.getPredictionError()
    if err > 1.0 then
        -- Large error means server disagreed significantly with our prediction
        ffe.log("Prediction error: " .. err)
    end
end
```

### 6. Lua: Server-Side Input Handling

```lua
local SPEED = 250.0
local INPUT_UP    = 1
local INPUT_DOWN  = 2
local INPUT_LEFT  = 4
local INPUT_RIGHT = 8

local players = {}  -- clientId -> entityId

function ffe.onInit()
    ffe.startServer(7777)
    ffe.setNetworkTickRate(20)

    ffe.onClientConnected(function(clientId)
        -- Spawn a player entity for the new client
        local id = ffe.createEntity()
        ffe.addTransform(id, 0, 0, 0, 1, 1)
        ffe.addPreviousTransform(id)
        ffe.addSprite(id, 1, 40, 40, 1.0, 0.4, 0.3, 1.0, 1)
        players[clientId] = id
    end)

    ffe.onClientDisconnected(function(clientId)
        if players[clientId] then
            ffe.destroyEntity(players[clientId])
            players[clientId] = nil
        end
    end)

    -- Handle input commands from clients (authoritative movement)
    ffe.onServerInput(function(clientId, inputTable)
        local entity = players[clientId]
        if not entity then return end

        local bits = inputTable.bits or 0
        local dt   = inputTable.dt or (1.0 / 60.0)

        local dx, dy = 0, 0
        if bits >= INPUT_UP    and (bits % (INPUT_UP * 2))    >= INPUT_UP    then dy = dy + SPEED * dt end
        if bits >= INPUT_DOWN  and (bits % (INPUT_DOWN * 2))  >= INPUT_DOWN  then dy = dy - SPEED * dt end
        if bits >= INPUT_LEFT  and (bits % (INPUT_LEFT * 2))  >= INPUT_LEFT  then dx = dx - SPEED * dt end
        if bits >= INPUT_RIGHT and (bits % (INPUT_RIGHT * 2)) >= INPUT_RIGHT then dx = dx + SPEED * dt end

        if dx ~= 0 or dy ~= 0 then
            local buf = {}
            if ffe.fillTransform(entity, buf) then
                ffe.setTransform(entity, buf.x + dx, buf.y + dy,
                                 buf.rotation, buf.scaleX, buf.scaleY)
            end
        end
    end)
end
```

### 7. C++: Client-Side Prediction with MoveFn (Full Reconciliation)

```cpp
#include "networking/prediction.h"
#include "networking/network_system.h"
using namespace ffe::networking;

// Movement function — applied both during prediction and reconciliation replay
void movePlayer(ffe::World& world, uint32_t entityId,
                const InputCommand& cmd, void*) {
    auto* t = world.getComponent<ffe::Transform>(entityId);
    if (!t) return;

    constexpr float SPEED = 250.0f;
    if (cmd.inputBits & 0x01) t->y += SPEED * cmd.dt;  // up
    if (cmd.inputBits & 0x02) t->y -= SPEED * cmd.dt;  // down
    if (cmd.inputBits & 0x04) t->x -= SPEED * cmd.dt;  // left
    if (cmd.inputBits & 0x08) t->x += SPEED * cmd.dt;  // right
}

// At startup (client side):
ffe::networking::setMovementFunction(movePlayer, nullptr);
ffe::networking::setLocalPlayer(playerEntityId);

// Each frame (client side):
InputCommand cmd;
cmd.inputBits = gatherInputBits();
// tickNumber and dt are filled automatically by sendInput
ffe::networking::sendInput(cmd);
```

### 8. C++: Low-Level Packet Serialisation

```cpp
#include "networking/packet.h"
using namespace ffe::networking;

// Send
uint8_t buf[MAX_PACKET_SIZE];
PacketWriter writer(buf, MAX_PACKET_SIZE);

PacketHeader hdr{};
hdr.type     = PacketType::INPUT;
hdr.channel  = 0;
hdr.sequence = nextSeq++;
writeHeader(writer, hdr);
writer.writeF32(playerX);
writer.writeF32(playerY);
writer.writeU8(buttonMask);

if (!writer.hasError()) {
    client.send(0, writer.data(), writer.bytesWritten(), true);
}

// Receive
void onReceive(const ReceivedPacket& pkt, void*) {
    PacketReader reader(pkt.data, pkt.dataLength);
    PacketHeader hdr;
    if (!readHeader(reader, hdr)) return;
    if (!isValidPacketType(static_cast<uint8_t>(hdr.type))) return;

    float x, y;
    uint8_t buttons;
    if (reader.readF32(x) && reader.readF32(y) && reader.readU8(buttons)) {
        handleInput(pkt.sender, x, y, buttons);
    }
}
```

## What Not to Do

### Do not skip `initNetworkSystem()` / `shutdownNetworkSystem()`

ENet requires global initialisation. The module-level `initNetworkSystem()` handles this plus default component registration. Calling `startServer` or `connectToServer` without it will fail or cause undefined behaviour. (Lua games get this automatically from the engine loop.)

### Do not hold `ReceivedPacket::data` beyond the callback

The `data` pointer in `ReceivedPacket` is a non-owning view into ENet's internal buffer. It is only valid for the duration of the receive callback. Copy it if you need the data later.

### Do not send packets larger than `MAX_PACKET_SIZE` (1200 bytes)

Packets are capped at 1200 bytes (MTU-safe). `PacketWriter::hasError()` returns true if you overflow. Split large data across multiple packets.

### Do not cast raw bytes to `PacketType` without validation

Always call `isValidPacketType(rawByte)` before casting a `uint8_t` to `PacketType`. Unvalidated casts produce undefined enum values.

### Do not ignore `PacketReader` / `PacketWriter` error flags

Every read/write returns `false` on failure. Once the error flag is set, all subsequent operations also fail. Always check `hasError()` or individual return values. Ignoring errors means processing garbage data.

### Do not use `readF32` output without checking the return value

`readF32` rejects NaN and Infinity for security. If it returns `false`, the float was non-finite or the buffer underflowed.

### Do not set callbacks after calling `start()` / `connect()`

Register callbacks before starting the transport. Setting them after may cause events to be missed during the handshake.

### Do not run both server and client simultaneously

The module-level API (`network_system.h`) enforces server XOR client. Calling `startServer` while a client is connected (or vice versa) will disconnect the current mode first. Design your game to be one or the other per process.

### Do not call `sendInput` on the server

`sendInput` is a client-only function. It records input in the prediction buffer and sends it to the server. On the server, use `onServerInput` to receive input from clients and apply authoritative movement directly. Calling `sendInput` on the server returns false and does nothing.

### Do not call `setLocalPlayer` before creating the entity

`setLocalPlayer` stores the entity ID for prediction. If the entity does not exist yet, reconciliation will fail silently. Always create the entity and add its Transform component first, then call `setLocalPlayer`.

### Do not manually reconcile predicted positions

The prediction system handles reconciliation automatically when server snapshots arrive. If the server position diverges from the predicted position by more than 0.1 units, the system snaps to the server position and replays buffered inputs. Do not write your own snap/lerp logic on top of this -- it will fight the built-in reconciliation.

### Do not use `std::function` or virtual dispatch for packet handling

Per CLAUDE.md Section 3, hot-path code avoids `std::function` (hidden heap allocations) and virtual calls. The API uses raw function pointers with `void* userData`. Follow the same pattern in game-level handlers.

### Do not process packets without rate-limit awareness (server side)

`ServerTransport` enforces rate limiting automatically (100 packets/sec, 64 KB/sec per connection). If you build custom server logic outside of `ServerTransport`, enforce rate limiting yourself.

## Tier Support

| Tier | Supported | Notes |
|------|-----------|-------|
| **RETRO** | No | ENet requires socket APIs that may not be available on minimal RETRO targets |
| **LEGACY** | **Yes (primary)** | No GPU involvement. Pure CPU + sockets. |
| **STANDARD** | Yes | Same implementation as LEGACY |
| **MODERN** | Yes | Same implementation as LEGACY |

The networking subsystem has no GPU involvement. Performance is bounded by socket I/O and packet processing, not rendering tier.
