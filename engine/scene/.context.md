# engine/scene/ -- System Context

## System Purpose

The scene subsystem provides two key features: a **scene graph** for parent-child entity hierarchies, and **serialisation/deserialisation** of ECS worlds to and from JSON.

The scene graph (`scene_graph.h`) manages parent-child relationships between entities using `Parent` and `Children` ECS components. It prevents circular references, supports ancestor queries, and provides root entity enumeration. The editor's hierarchy tree panel and reparent command are built on top of it.

The serialiser (`scene_serialiser.h`) enables saving and loading entire scenes (all entities and their components) to disk or to in-memory strings. It is used by the editor for scene persistence and by the engine for runtime scene loading. The serialiser enforces security constraints: file size limits, entity count caps, float validation, and path traversal rejection. It uses a versioned format so future changes can be handled gracefully.

## Public API

### Scene Graph (`engine/scene/scene_graph.h`, namespace `ffe::scene`)

Manages parent-child entity relationships. Uses the ECS `Parent` and `Children` components to form a hierarchy tree. Prevents circular references and caps traversal depth at 64 levels.

#### Constants

| Name | Type | Value | Description |
|------|------|-------|-------------|
| `MAX_HIERARCHY_DEPTH` | `uint32_t` | `64` | Maximum ancestor traversal depth. Prevents infinite loops on corrupted data |

#### Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `setParent` | `bool setParent(World& world, entt::entity child, entt::entity parent)` | Set parent-child relationship. Removes any existing parent first. Returns false if it would create a cycle (parent is descendant of child) or if child == parent. Also returns false if parent's children array is full (32 max) |
| `removeParent` | `void removeParent(World& world, entt::entity child)` | Remove parent from entity (makes it a root). No-op if already a root |
| `getParent` | `entt::entity getParent(const World& world, entt::entity entity)` | Get parent entity. Returns `entt::null` if root |
| `getChildren` | `const entt::entity* getChildren(const World& world, entt::entity entity, uint32_t& count)` | Get pointer to children array and set count. Returns nullptr and count=0 if entity has no children |
| `isRoot` | `bool isRoot(const World& world, entt::entity entity)` | True if the entity has no Parent component |
| `isAncestor` | `bool isAncestor(const World& world, entt::entity entity, entt::entity ancestor)` | Walk up the parent chain to check if `ancestor` is an ancestor of `entity`. Returns false if entity == ancestor |
| `getRootEntities` | `uint32_t getRootEntities(const World& world, entt::entity* output, uint32_t maxCount)` | Collect all root entities (no Parent component) into `output`. Returns number written, up to `maxCount` |

#### ECS Components Used

| Component | Fields | Description |
|-----------|--------|-------------|
| `Parent` | `entt::entity parent` | Stores the parent entity handle. Entities without this component are roots |
| `Children` | `entt::entity children[32]`, `uint32_t count` | Fixed-capacity array of child entity handles. Created/removed automatically by `setParent`/`removeParent` |

### Scene Serialiser (`engine/scene/scene_serialiser.h`, namespace `ffe::scene`)

All functions are in namespace `ffe::scene` (header: `engine/scene/scene_serialiser.h`).

### Constants

| Name | Type | Value | Description |
|------|------|-------|-------------|
| `MAX_SERIALISED_ENTITIES` | `uint32_t` | `10000` | Maximum entities in a scene file. Prevents unbounded memory allocation from malicious or corrupt files |
| `MAX_SCENE_FILE_SIZE` | `size_t` | `64 MB` | Maximum file size accepted by `loadScene`. Prevents memory exhaustion during JSON parsing |
| `SCENE_FORMAT_VERSION` | `int32_t` | `1` | Current format version. Incremented when the format changes |

### Functions

| Function | Signature | Description |
|----------|-----------|-------------|
| `saveScene` | `bool saveScene(const World& world, const std::string& path)` | Serialise all entities to a JSON file. Returns true on success. Rejects paths containing `..` |
| `loadScene` | `bool loadScene(World& world, const std::string& path)` | Load entities from a JSON file into the world. Does NOT clear existing entities first. Returns true on success. Rejects paths containing `..` and files exceeding `MAX_SCENE_FILE_SIZE` |
| `serialiseToJson` | `std::string serialiseToJson(const World& world)` | Serialise all entities to a JSON string. Useful for testing, clipboard, and network transfer. No file I/O |
| `deserialiseFromJson` | `bool deserialiseFromJson(World& world, const std::string& json)` | Deserialise entities from a JSON string into the world. Returns true on success. Rejects strings exceeding `MAX_SCENE_FILE_SIZE` |

### Serialised Component Types

The following components are serialised and deserialised:

| Component | Serialised Fields |
|-----------|-------------------|
| `Name` | `name` (string, truncated to 63 chars) |
| `Transform` | `x`, `y`, `z`, `rotation`, `scaleX`, `scaleY` |
| `Transform3D` | `px`, `py`, `pz`, `rw`, `rx`, `ry`, `rz`, `sx`, `sy`, `sz` |
| `Parent` | `parent` (index into the entity array, not raw entity ID) |

**Children** components are NOT serialised directly. They are rebuilt from Parent references during deserialisation.

Components not in this list (Sprite, Material3D, Mesh, ParticleEmitter, Tilemap, etc.) are not yet serialised. Entities with unsupported components will lose that data on save/load.

### JSON Format

```json
{
  "version": 1,
  "entities": [
    {
      "name": "Player",
      "components": {
        "Name": { "name": "Player" },
        "Transform": {
          "x": 0.0, "y": 0.0, "z": 0.0,
          "rotation": 0.0,
          "scaleX": 1.0, "scaleY": 1.0
        },
        "Parent": { "parent": 0 }
      }
    }
  ]
}
```

- `version` (required): Integer matching `SCENE_FORMAT_VERSION`. Files with mismatched versions are rejected.
- `entities` (required): Array of entity objects.
- Each entity has a `components` object with component type names as keys.
- `Parent.parent` is an index into the `entities` array (not a raw entity ID). This allows entity references to survive serialisation.
- Entity names appear both in `components.Name.name` and at the top-level `name` field for readability.

## Common Usage Patterns

### 1. Build a parent-child hierarchy

```cpp
#include "scene/scene_graph.h"

auto parent = world.createEntity();
auto child1 = world.createEntity();
auto child2 = world.createEntity();

ffe::scene::setParent(world, child1, parent);  // true
ffe::scene::setParent(world, child2, parent);  // true

// Query
ffe::scene::getParent(world, child1);  // == parent
ffe::scene::isRoot(world, parent);     // true
ffe::scene::isRoot(world, child1);     // false

uint32_t count = 0;
const auto* kids = ffe::scene::getChildren(world, parent, count);
// count == 2, kids[0] and kids[1] are child1/child2 (order not guaranteed)
```

### 2. Reparent an entity (move in the tree)

```cpp
#include "scene/scene_graph.h"

// Move child1 from parent to grandparent — old parent detached automatically
ffe::scene::setParent(world, child1, grandparent);

// Make child1 a root entity (detach from any parent)
ffe::scene::removeParent(world, child1);
```

### 3. Check for circular references

```cpp
#include "scene/scene_graph.h"

// isAncestor prevents cycles — setParent checks this internally
bool wouldCycle = ffe::scene::isAncestor(world, parent, child);
// If child is an ancestor of parent, setParent(world, parent, child) returns false
```

### 4. Enumerate all root entities

```cpp
#include "scene/scene_graph.h"

entt::entity roots[256];
uint32_t rootCount = ffe::scene::getRootEntities(world, roots, 256);
for (uint32_t i = 0; i < rootCount; ++i) {
    // roots[i] is a top-level entity with no parent
}
```

### 5. Save and load a scene file (serialiser)

```cpp
#include "scene/scene_serialiser.h"

// Save
ffe::scene::saveScene(world, "assets/scenes/level1.json");

// Load (clear first if you want a fresh scene)
world.clearAllEntities();
ffe::scene::loadScene(world, "assets/scenes/level1.json");
```

### 6. Serialise to/from string (testing or clipboard)

```cpp
#include "scene/scene_serialiser.h"

// Serialise to string
std::string json = ffe::scene::serialiseToJson(world);

// Deserialise from string (additive -- does not clear existing entities)
ffe::scene::deserialiseFromJson(world, json);
```

### 7. Duplicate a scene (copy via serialisation)

```cpp
// Snapshot the current state
std::string snapshot = ffe::scene::serialiseToJson(world);

// Clear and reload
world.clearAllEntities();
ffe::scene::deserialiseFromJson(world, snapshot);
```

### 8. Additive loading (merging scenes)

```cpp
// Load base level
world.clearAllEntities();
ffe::scene::loadScene(world, "assets/scenes/base.json");

// Add props on top (does NOT clear -- entities are added)
ffe::scene::loadScene(world, "assets/scenes/props.json");
```

### 9. Using in the editor with undo

```cpp
// Before a destructive operation, snapshot for undo
std::string before = ffe::scene::serialiseToJson(world);

// ... do something destructive ...

// To undo: clear and restore
world.clearAllEntities();
ffe::scene::deserialiseFromJson(world, before);
```

## What Not To Do

### Do not manually emplace Parent/Children components

Use `setParent()` and `removeParent()` instead. These functions maintain the bidirectional relationship (Parent on child, Children on parent), check for cycles, and clean up empty Children components. Direct `registry().emplace<Parent>()` bypasses all of this and will corrupt the hierarchy.

```cpp
// BAD — Children list not updated, no cycle check
world.registry().emplace<Parent>(child, Parent{parent});

// GOOD
ffe::scene::setParent(world, child, parent);
```

### Do not exceed 32 children per entity

The `Children` component uses a fixed-size array of 32 entries. `setParent()` returns false if the parent's children array is full. If you need more children, restructure the hierarchy with intermediate grouping entities.

### Do not assume child ordering is stable

`removeChildFromParent` uses swap-with-last removal for O(1) performance. The order of children in the array may change after removals. If you need stable ordering, sort by a separate component (e.g., a sibling index).

### Do not assume loadScene clears the world

`loadScene` and `deserialiseFromJson` are additive. They create new entities without removing existing ones. Call `world.clearAllEntities()` first if you want a clean load.

```cpp
// BAD -- old entities remain, scene is a mess
ffe::scene::loadScene(world, "level2.json");

// GOOD -- clean slate
world.clearAllEntities();
ffe::scene::loadScene(world, "level2.json");
```

### Do not use paths with `..` components

Both `saveScene` and `loadScene` reject any path containing `..` as a path traversal defence. Use absolute paths or simple relative paths without parent directory references.

```cpp
// BAD -- rejected
ffe::scene::saveScene(world, "../../../etc/passwd");

// GOOD
ffe::scene::saveScene(world, "assets/scenes/level1.json");
```

### Do not serialise scenes larger than 10,000 entities without adjusting the cap

`MAX_SERIALISED_ENTITIES` is 10,000. Scenes with more entities will fail to deserialise. If you genuinely need larger scenes, the cap must be increased at the source level.

### Do not rely on entity IDs surviving save/load

Entity IDs are not preserved across serialisation. After loading, entities receive new IDs. Parent references are serialised as array indices and resolved to the new IDs during deserialisation. Any external references to entity IDs (stored in game code, Lua scripts, etc.) will be stale after a load.

### Do not expect non-finite floats to round-trip

The deserialiser validates all float fields and rejects entities containing NaN or infinity. If your scene data contains non-finite values, the load will fail.

### Do not expect Sprite, Material3D, Mesh, or Tilemap to be saved

Only Name, Transform, Transform3D, and Parent are currently serialised. Visual components (textures, meshes, materials) are not yet persisted. This will be extended in future sessions.

## Tier Support

The scene serialiser is tier-agnostic. It operates on CPU-side ECS data with no GPU dependency. It works identically on all tiers (RETRO through MODERN).

**Performance note:** Serialisation and deserialisation use `nlohmann/json`, which allocates on the heap. These are cold-path operations (user-triggered save/load, scene transitions) and are not called per-frame. The `MAX_SCENE_FILE_SIZE` cap (64 MB) prevents memory exhaustion.
