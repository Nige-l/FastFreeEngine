# editor/build/ -- Build Pipeline

Written by api-designer, Session 56.

## System Purpose

The build pipeline exports a standalone game package from the editor. Given a `BuildConfig` (project name, entry scene, asset/script directories), `exportGame()` assembles a self-contained directory containing the runtime binary, all assets, scene files, scripts, and a generated `main.lua` that loads the entry scene via `ffe.loadSceneJSON()`. The exported game runs independently of the editor.

---

## Public API

Namespace: `ffe::editor`

### BuildConfig (`build_config.h`)

Fixed-size configuration struct. No heap allocations -- uses `char[]` buffers.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `projectName` | `char[64]` | `"MyGame"` | Game name; used as output subdirectory name and renamed binary |
| `entryScene` | `char[256]` | `""` | Relative path to the entry scene JSON file |
| `outputDir` | `char[256]` | `"export/"` | Root directory for export output |
| `assetDirs` | `char[4][256]` | `{"assets/", "textures/", "", ""}` | Asset directories to copy (up to 4) |
| `scriptDirs` | `char[2][256]` | `{"scripts/", ""}` | Script directories to copy (up to 2) |
| `assetDirCount` | `int` | `2` | Number of active asset directory entries |
| `scriptDirCount` | `int` | `1` | Number of active script directory entries |

| Function | Signature | Description |
|----------|-----------|-------------|
| `saveBuildConfig` | `bool saveBuildConfig(const BuildConfig& config, const std::string& path)` | Serialise config to JSON. Returns true on success |
| `loadBuildConfig` | `bool loadBuildConfig(BuildConfig& config, const std::string& path)` | Deserialise config from JSON. Config unchanged on failure. Returns true on success |

### Exporter (`exporter.h`)

| Type | Fields | Description |
|------|--------|-------------|
| `ExportResult` | `bool success`, `std::string errorMessage`, `std::string outputPath` | Result of an export operation |

| Function | Signature | Description |
|----------|-----------|-------------|
| `exportGame` | `ExportResult exportGame(const BuildConfig& config, const std::string& runtimeBinaryPath)` | Export a standalone game package. Creates `<outputDir>/<projectName>/`, copies the runtime binary (renamed to project name), copies asset dirs into `assets/`, copies scene files into `scenes/`, copies script dirs into `scripts/`, and generates `main.lua` |

**Export directory layout:**
```
<outputDir>/<projectName>/
    <projectName>          -- runtime binary (copied + renamed)
    main.lua               -- generated entry point
    assets/                -- copied from assetDirs
    scenes/                -- copied scene JSON files
    scripts/               -- copied from scriptDirs
```

**Generated `main.lua`** calls `ffe.loadSceneJSON("<entryScene>")` to load the first scene.

## Common Usage Patterns

### 1. Export a game programmatically

```cpp
#include "build/build_config.h"
#include "build/exporter.h"

ffe::editor::BuildConfig config{};
std::strncpy(config.projectName, "MyPlatformer", 63);
std::strncpy(config.entryScene, "scenes/level1.json", 255);
std::strncpy(config.outputDir, "export/", 255);

auto result = ffe::editor::exportGame(config, "/path/to/ffe_runtime");
if (result.success) {
    // Game exported to result.outputPath
} else {
    // Check result.errorMessage
}
```

### 2. Persist build settings across sessions

```cpp
ffe::editor::BuildConfig config{};
// ... configure ...
ffe::editor::saveBuildConfig(config, "build_config.json");

// Later:
ffe::editor::BuildConfig loaded{};
ffe::editor::loadBuildConfig(loaded, "build_config.json");
```

### 3. Use via the BuildPanel UI

The `BuildPanel` provides an ImGui interface for the above. Call `panel.draw()` each frame. The panel exposes all `BuildConfig` fields as editable widgets and an "Export Game" button that calls `exportGame()`.

## What Not To Do

### Do not export without setting entryScene

The default `entryScene` is empty. The generated `main.lua` will call `ffe.loadSceneJSON("")`, which fails at runtime.

### Do not assume the runtime binary is on PATH

`exportGame` copies the file at `runtimeBinaryPath` verbatim. It does not search `PATH`. Provide an absolute or correct relative path to the pre-built `ffe_runtime` binary.

### Do not exceed the fixed buffer sizes

`projectName` is 64 chars, paths are 256 chars. Longer strings will be silently truncated by `strncpy`. Keep project names and paths within these limits.

### Do not call exportGame from a hot path

`exportGame` performs filesystem operations (directory creation, file copying). It is a cold-path operation triggered by user action, not a per-frame call.

---

## Tier Support

The build pipeline is editor-only infrastructure. It runs at editor time, not game runtime. No tier constraints apply.

## Dependencies

| Dependency | What For |
|------------|----------|
| **\<filesystem\>** | Directory creation, file copying during export |
| **FFE scene serialiser** | `ffe::scene::loadScene()` is what the exported runtime calls via `ffe.loadSceneJSON()` |
| **nlohmann/json** (or equivalent) | BuildConfig serialisation/deserialisation |
